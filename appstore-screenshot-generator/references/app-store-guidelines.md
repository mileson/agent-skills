# App Store 截图设计指南

> 参考文档：Apple 官方规范与行业最佳实践汇总

## 1. 截图尺寸规格

### iPhone 必需尺寸

| 设备 | 显示屏 | 像素尺寸 | 必需性 |
|-----|-------|---------|-------|
| iPhone 16/15 Pro Max | 6.7" | 1320×2868 / 1290×2796 | ✅ 必需 |
| iPhone 14/13/12 Pro Max | 6.5" | 1284×2778 | ✅ 必需 |
| iPhone 8 Plus | 5.5" | 1242×2208 | ⚠️ 推荐 |

### iPad 尺寸（如支持）

| 设备 | 显示屏 | 像素尺寸 |
|-----|-------|---------|
| iPad Pro 12.9" | 12.9" | 2048×2732 |
| iPad Pro 11" | 11" | 1668×2388 |

### 重要规则

- 最少需要 **1 张截图**，最多 **10 张**
- 首选提供 6.7" 和 6.5" 两种尺寸
- 其他尺寸会自动缩放适配
- 截图必须是 **PNG** 或 **JPEG** 格式
- 不允许有透明背景

## 2. 设计最佳实践

### 2.1 首图原则 (First Impression)

首张截图决定 **70%+ 的下载转化率**：

- ✅ 展示应用核心价值
- ✅ 使用最吸引人的功能画面
- ✅ 文案简洁有力（2-6 个词）
- ❌ 不要放次要功能
- ❌ 不要文字过多

### 2.2 视觉层次

```
┌────────────────────────────────┐
│  [主标题] - 最大、最醒目       │
│                                │
│     ┌──────────────────┐       │
│     │                  │       │
│     │    设备截图       │       │
│     │   (核心内容)     │       │
│     │                  │       │
│     └──────────────────┘       │
│                                │
│  [副标题/功能说明] - 较小      │
└────────────────────────────────┘
```

### 2.3 设备 Mockup 使用

**推荐做法：**
- 使用最新设备型号（iPhone 15 Pro 系列）
- 设备倾斜 5-15° 增加动感
- 保持设备边框完整
- 屏幕内容清晰可读

**避免：**
- 使用过时设备型号
- 设备过小看不清内容
- 设备裁剪不当

### 2.4 背景设计

| 风格 | 适用类型 | 示例颜色 |
|-----|---------|---------|
| 渐变 | 通用 | #667eea → #764ba2 |
| 纯色 | 工具/效率 | #FAFAF8 (浅) / #1C1C1E (深) |
| 深色 | 摄影/游戏 | #000000 / #1A1A2E |
| 活力色 | 社交/生活 | #FF6B6B → #9B59B6 |

### 2.5 文案撰写

**好的文案：**
- "Capture Perfect Moments" ✅
- "Track Every Rep" ✅
- "Your Money, Simplified" ✅

**差的文案：**
- "The best photo app with many features" ❌ (太长)
- "Download now!" ❌ (无信息量)
- "Version 2.0" ❌ (用户不关心)

**本地化要点：**
- 中文文案更短更有力
- 避免直译，要本土化表达
- 考虑文字长度对布局影响

## 3. 按应用类型的设计建议

### 3.1 摄影/相机类

- **色调**: 深色背景，突出图片质量
- **元素**: 镜头光晕、光线效果
- **文案**: 强调专业、创意、简单
- **示例**: "Pro-Level Photos" / "专业级照片"

### 3.2 健身/健康类

- **色调**: 深色 + 能量色（绿/橙）
- **元素**: 动态线条、进度环、运动图标
- **文案**: 强调改变、追踪、目标
- **示例**: "Transform Your Body" / "改变你的身体"

### 3.3 生产力/工具类

- **色调**: 浅色、干净、简约
- **元素**: 最小化装饰，留白为主
- **文案**: 强调简化、效率、掌控
- **示例**: "Simplify Your Day" / "简化你的一天"

### 3.4 社交/生活类

- **色调**: 温暖渐变、活力色彩
- **元素**: 多设备、连接线、表情符号
- **文案**: 强调连接、分享、真实
- **示例**: "Connect Authentically" / "真实连接"

### 3.5 游戏类

- **色调**: 匹配游戏风格
- **元素**: 游戏角色、特效、奖励徽章
- **文案**: 强调乐趣、挑战、免费
- **示例**: "Epic Adventure Awaits" / "史诗冒险等你来"

### 3.6 金融/商务类

- **色调**: 深蓝、深绿 + 金色点缀
- **元素**: 安全徽章、数据图表
- **文案**: 强调安全、简单、掌控
- **示例**: "Your Money, Simplified" / "你的财务，简单掌控"

## 4. A/B 测试建议

### 测试要素

1. **标题文案** - 测试不同的价值主张
2. **背景颜色** - 浅色 vs 深色
3. **设备角度** - 直立 vs 倾斜
4. **功能展示** - 不同的核心功能
5. **截图顺序** - 不同的故事流程

### 测试工具

- App Store Connect 原生 A/B 测试
- SplitMetrics
- StoreMaven

## 5. 常见错误

### 避免这些问题：

1. ❌ 文字太小看不清
2. ❌ 截图内容与功能不符
3. ❌ 设备遮挡重要信息
4. ❌ 背景过于花哨分散注意力
5. ❌ 所有截图风格不统一
6. ❌ 忽视本地化差异
7. ❌ 使用低分辨率素材
