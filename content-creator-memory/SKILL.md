# content-creator-memory

> **关联 Skill**: 这是 [content-creator](~/.cursor/skills/content-creator/) 的配套组件，提供内容记忆和智能检索功能。

## Overview

本 skill 为 content-creator 提供高效的内容检索能力，基于 80/20 记忆策略：
- **80% 权重**: 自有内容（own_works）- 保持个人风格和表达习惯
- **20% 权重**: 标杆案例（reference_examples）- 借鉴爆款技巧和结构模式

**设计理念**: 仅通过 content-creator skill 调用，不直接响应用户请求。

---

## 核心功能

### 1. 搜索自有内容

**用途**: 检索历史作品，用于选题策划和风格校准

**输入参数**:
- 搜索查询词（必需）
- 过滤条件（可选）：目标受众、技术栈、内容类型、语调等
- 返回数量（默认: 5）
- 最低质量分数（默认: 6.0）

**输出**: 相关内容列表，包含标题、摘要、标签、质量分数、可复用元素等

**使用场景**:
- 选题策划：检索相似主题的成功案例
- 标题大纲：提取个人风格的表达模式

---

### 2. 搜索标杆案例

**用途**: 检索标杆案例的标题钩子、结构模式等技巧

**输入参数**:
- 搜索查询词（必需）
- 过滤条件（可选）：平台、作者等
- 返回数量（默认: 3）

**输出**: 相关案例列表，包含标题、作者、平台、核心技巧等信息

**使用场景**:
- 标题大纲：借鉴爆款标题结构
- 写作剧本：参考叙事节奏和段落组织

---

### 3. 重建索引

**用途**: 全量重建索引文件（初始化或大批量更新后）

**输入参数**:
- 自有内容目录（Markdown 文件所在位置）
- 标杆案例目录（按作者分类的案例）
- 索引文件输出目录

**输出**: 生成索引文件（自有作品、标杆案例、搜索索引）

**使用时机**:
- 初始化索引（首次使用）
- 批量添加内容后（10篇以上）
- 索引损坏需修复

---

### 4. 增量添加内容

**用途**: 添加单篇新内容到索引（无需全量重建）

**输入参数**:
- 新文章的 Markdown 文件路径（必需）
- 是否自动提取元数据（可选）

**输出**: 更新索引文件，保留现有数据

**使用时机**:
- 每次创作完成新文章后立即更新索引

---

### 索引生成流程

1. 扫描 Markdown 文件（递归）
2. 提取元数据（YAML frontmatter）
3. 构建 TF-IDF 索引（基于标题、摘要、标签）
4. 保存 JSON 索引文件

---

### 搜索算法

- **TF-IDF 检索**：计算相关性分数
- **过滤条件应用**：根据元数据筛选
- **质量排序**：优先返回高质量内容

## 性能指标

- **搜索响应**: 1-3ms（预加载索引）
- **索引大小**: ~222KB（104 篇内容）
- **内存占用**: ~2MB（索引加载后）
- **支持内容量**: 1000+ 篇（无性能下降）
