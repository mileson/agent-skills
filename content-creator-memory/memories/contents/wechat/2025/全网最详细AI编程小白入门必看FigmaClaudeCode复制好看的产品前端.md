一、AI 编程复刻核心思路
=============

这套 AI 编程复刻工作流的核心思路是通过 Figma 将网页转化为 Figma 设计稿，然后借助 AI 编程复刻提示词即可完成高还原度。

  

复刻的核心流程如下，关键的操作步骤为需要将网站导出为 Figma 设计稿，然后在 AI 编程工具内通过文章内提供的AI 编程提示词 + Figma 设计稿地址的流程，就可以复刻出来高还原度的产品了。

如果你也希望掌握一套成熟可行的 AI 编程复刻方法，那么，请跟着我们接下来保姆级的教程，来学习这个超级有用的 AI 编程技巧，让我们先从配置 AI 编程复刻工作流开始：

  

二、配置 AI 编程复刻工作流
===============

接下来，我将为你详细我将为你介绍针这套 AI 编程复刻工作流如何搭建，希望你能跟着详细的步骤，即使是新手也可以从到 0 到 1 完成搭建：

  

1\. 安装 Figma 客户端，并注册 Figma 账号
-----------------------------

如果你还没有下载并注册过 Figma ，可以通过如下详细步骤，完成从账号注册到安装、初始化的全流程（如果你已经下载客户端并注册账号，可以跳过该步骤，从第二步开始）：

2\. 将复刻的产品导出为 Figma 设计稿
-----------------------

在完成 Figma 的初始化后，请通过如下步骤完成网页转 Figma 设计稿的插件的安装。并且，你将学会一套将网站自动转化为 Figma 设计稿的工作流。

  

### 2.1 安装 Figma 插件

通过下图流程，我们将完成 Figma 客户端内的插件「 html.to.design」与 Chrome浏览器插件的安装：

两个「 html.to.design」插件主要的用途如下：

*   Chrome 浏览器插件：主要用于将网页内容导出
*   Figma 客户端插件：主要用于将导出的内容同步到 Figma 内生成设计稿

  

### 2.2 导出复刻的网站内容

在前面步骤，我们已经完成了 Chrome 插件「 html.to.design」的安装。接下来，只需要前往需要复刻的网站页面，如下图流程所示，打开插件并导出网站内容为本地文件，用于后续导入 Figma 使用。

  

本环节核心的步骤为：

*   打开 Chrome 插件「 html.to.design」
*   确认导出的设置
*   选择导出的方式（整页导出或点选部分区域导出）
*   导出完成（下载为 h2d 文件）

通过如上步骤，我们就可以将网站的内容直接导出，用于后续我们将一键导入到 Figma 客户端内。

  

### 2.3 将网站转化为 Figma 设计稿

在这个环节，我们将实现将导出本地的网站内容，一键导入到 Figma 设计稿，让我们跟着如下步骤一起来完成这个操作。

  

本环节核心的步骤为：

*   在 Figma 客户端的画布界面，打开 Figma 客户端插件「 html.to.design」
*   导入下载的 h2d 网站文件
*   确认导入的设置
*   导入完成（画布上将显示出导入的网站界面）

根据上述步骤，我们就完成了从网站内容导出、导入到 Figma 的全流程，接下来，我们就将学到如何配置 AI 编程工具，实现 AI 可以读取到 Figma 设计稿进行高还原度的复刻。

  

3\. 配置 AI 编程工具
--------------

在正式开始 AI 编程之前，我们需要对 AI 编程工具进行简单的配置，我将从 0 到 1 教会你如何配置，帮助你更好的掌握入门 AI 编程与复刻操作。

  

### 3.1 进入项目文件夹

如果你准备开始复刻网站了，并且还没有创建项目文件夹，请根据下述步骤进行操作，创建完成文件夹并选择你的 AI 编程工具进入项目文件夹内。

特别说明，前面我们说到本次的复刻流程使用的是智谱的 GLM-4.6 AI 编程模型，所以，Cursor 跟 Claude Code 只是用于使用该 AI 编程模型，不需要开通会员。接下来，我将带着你一步步配置完成 GLM-4.6 这款 AI 编程模型（仅需 20 元/月）。

  

### 3.2 配置智谱 GLM-4.6 模型

#### 3.2.1 开通智谱 Coding Plan

在配置 智谱 GLM-4.6 AI 编程模型之前，我们需要通过智谱网站（bigmodel点cn/glm-coding）注册并开通 Coding Plan 套餐，选择月度套餐就行，仅需 20 元（如下图），点击\[特惠订阅\] 跟随指引即可完成购买开通。

#### 

#### 3.2.2 获取 GLM API Key 信息

开通 Coding Plan 后，请通过如下的流程，在智谱后台（bigmodel点cn/usercenter/proj-mgmt/apikeys）创建并复制 API Key 用于配置 AI 编程工具，注意需要要先登录账号才能进入后台：

最后，我们通过如下流程，将 API Key 配置到 Cursor 或 Claude Code AI 编程工具内，就可以直接使用智谱 GLM 进行高性价比的 AI 编程了：

  

#### 3.2.3 在 AI 编程工具配置 GLM-4.6

通过如下流程，我们将 3.2.2 步骤处获取到的 API Key 配置到 AI 编程工具内，请根据使用的 AI 编程工具（Cursor 或 Claude Code），选择合适的配置流程：

（1） 配置 GLM-4.6 （ Cursor 版 ）

  

上述流程中，涉及到的 Base URL 信息，请复制如下的内容进行配置：

    # Cursor 配置 GLM-4.6 模型信息

    Override OpenAI Base URL 信息：https://open.bigmodel.cn/api/coding/paas/v4

  

  

（2）配置 GLM-4.6 （ Claude Code 版 ）

上述流程中，涉及到的指令信息，请根据你的操作系统，选择合适的指令并复制后输入到终端命令行 cmd 内进行执行：

    # Claude Code 配置 GLM-4.6 模型指令

    ## macOS 或 Linux 系统（在终端命令行 cmd 内输入并回车键↩︎发送如下指令）

    curl -O "https://cdn.bigmodel.cn/install/claude_code_env.sh" && bash ./claude_code_env.sh

    ## Windows 系统（在终端命令行 cmd 内依次输入并回车键↩︎发送如下指令）

    setx ANTHROPIC_AUTH_TOKEN [替换为你的 API Key]

    setx ANTHROPIC_BASE_URL https://open.bigmodel.cn/api/anthropic

    setx CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC 1

  

### 3.3 Figma MCP

在开始正式的 AI 编程复刻之前，我们还需要做最后一个环节的配置，那就是为 AI 编程工具配置 Figma MCP ，我们可以将 Figma MCP 理解为 AI 编程工具的插件，配置后，AI 就可以通过这个插件读取到我们提示词中添加的 Figma 设计稿地址内的具体内容啦。

  

让我们根据流程一步步完成最后的配置，配置完成后，我们就可以开始 AI 编程复刻网站了：

  

#### 3.3.1 配置 Figma MCP（Cursor 版）

Figma MCP 配置信息如下，请复制后进行配置：

    # Figma MCP 配置信息

    "figma": {    

        "url": "https://mcp.figma.com/mcp"

    }

  

#### 3.3.2 配置 Figma MCP（Claude Code 版）

Figma MCP Claude Code 指令如下，请复制后在终端内执行，即可完成自动安装 Figma MCP：

    # 配置 Figma MCP 指令（在终端命令行 cmd 内依次输入并回车键↩︎发送如下指令）

    claude mcp add --transport http figma https://mcp.figma.com/mcp

  

  

4\. 通过 AI 编程复刻网站
----------------

### 4.1 输入提示词并开始复刻网站

AI 编程复刻网站提示词如下，请你根据上述流程的第一步骤，使用获取的 Figma 设计稿地址替换提示词内的 \[替换为 Figma 设计稿网址\] 内容，然后将完整的提示词复制并发送给 AI 编程工具（Cursor 或 Claude Code），即可完成网站的复刻：

    # AI 复刻网站提示词（Web网站产品）请你作为一名首席前端架构师，基于 [替换为 Figma 设计稿网址] 对这个网站进行 100% 高保真复刻。你的任务是：分析、声明、然后执行。禁止任何形式的“臆造”、“简化”或“遗漏”。【第一步：强制蓝图分析（你必须先完成此步）】在你编写任何代码之前，请首先用文字输出你的“施工蓝图”。此蓝图必须包含（但不限于）：1.顶层布局结构： 明确说明你识别出的顶层布局（例如：`[固定侧边栏 | 头部 | 内容区]`，或 `[头部 | 主体内容]` 等）。2.复杂模块识别： 列出你识别出的所有主要的 UI 模块（例如：统计卡片、数据表格、项目网格等）。3.关键计数声明： 自动清点 (Count) 页面上的关键可重复元素，并在此处声明其数量（例如：`"侧边栏有 N 个菜单项"`, `"项目网格是 N x N 布局，共 N 个卡片"`）。4.关键视觉识别： 声明你识别出的关键视觉元素（例如：`"图表类型是折线图"`，`"图标类型是品牌 Logo"`）。【第二步：编码执行（严格遵守你的蓝图）】蓝图分析完成后，你必须严格按照你自己在第一步中声明的蓝图开始编码，并遵循以下“零容忍”规则：1.【零容忍：布局与计数】 严格执行你声明的“顶层布局”和“关键计数”。如果蓝图说有 9 个卡片，就绝不能只做 6 个。2.【零容忍：图标与 Logo】 禁止用“通用图标”替换“品牌 Logo”！ 如果设计稿中是品牌 Logo (如 Apple, Google, Svelte)，你必须 1:1 还原它们，而不是使用通用图标 (如 `</>`)。3.【零容忍：图表】 禁止替换图表类型！ 同时，必须视觉上复刻图表的“关键视觉特征”（例如：明显的尖峰、低谷或曲线走势），禁止只画一条平坦的线。【第三步：最终自检】1. 开发完成后，请自动对照你在“第一步”中声明的蓝图和原始设计稿进行自检，确保没有任何声明的要点被遗漏。2. 开发完成后，请自动运行并告诉我测试地址。ultralthink
