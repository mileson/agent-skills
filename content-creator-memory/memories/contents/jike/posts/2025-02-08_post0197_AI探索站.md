# 即刻动态 197

**发布时间:** 2025-02-08  
**圈子:** AI探索站

---

刚才用了下字节的 Trae ，新用户上手工具的体验确实比 Cursor 好一些，当然也是踩在 Cursor 的肩膀上

因为就体验了几分钟，也发现了一些局限性吧，不一定对，给大家参考下：

《不足》
一、模型层面
1️⃣ 目前只支持 Claude 模型 （Chat 交流模式下额外有GPT-4o），没有其他模型选择

二、AI编程操作层面
1️⃣ 不支持 Agent 模式，像早期的 Cursor 一样需要单独对话
2️⃣ 直接引入对话的操作方式不够直接，只能添加图片，不支持把文件、文件夹这些拖到对话框，需要# 符号才能选择进去

《优点》
1️⃣ 工具操作体验舒适，界面布局更直观，Cursor 藏了挺多功能在看不到的地方，Trae 基本在一个屏幕内都可以找到，细节打磨比较完善，有一定工具的美感
2️⃣ 原生支持简体中文，不需要像 Cursor 一样安装插件才行
3️⃣ 目前最适合 AI编程的 Claude 模型无限制使用，不需要付费（未来不知道）
4️⃣ Builder 模式，听说可以从 0 到 1 创建项目，目前没有深度使用暂时不支持，我用了默认的提示词做贪吃蛇，就创建了一个文件，也不是一个项目
5️⃣ 支持 Web 预览模式，对于写网站不需要打开单独页面，直接编辑器里面可以预览，好像这个通过插件也能解决，如果网站编程情况比较多的话，确实比较友好

总结，目前应该是追赶 Cursor 的状态，如果之前已经在用 Cursor 比较多的创造者们，乃至主力是 agent 模式的，建议可以再观望下 Trae 的发展，简单的产品就少数几个文件的 Trae 应该是没问题的，复杂一些的，建议还是继续使用 Cursor。
