# 即刻动态 342

**发布时间:** 2025-05-06  
**圈子:** AI探索站

---

对于 Vibe Coding 不止要学编程这个理解是一样的，不过我感觉这个过程应该是循序渐进的，以往传统的编程方式是需要先学习再编码，门槛太高导致沦为少数人的工具。

Vibe Coding 提供了一个极低的门槛，让大部分不懂技术的泛互联网人群首次具备了创造软件产品的能力。

一句话就能实现一个极简的贪吃蛇、计算器等小产品，给了作为学生入门的人群一个快速正反馈的 aha moment，不像早期作为软件工程学生的我，初入学需要不断练习好几周的 hello world，漫长的学习曲线，也熬走了不少“耐不住寂寞”的人。

打好基本功固然好，但是也不妨碍基于兴趣、目标导向的边练边学，当然，这里会存在会不会学不精的问题。

但是，我认为做出来比起学起来再做出来，我更愿意看到越来越多的人：在一次次的实践中跌倒并吸取教训，掌握基于自己边界内的能力，用到了再学，这是 Vibe Coding 或者说 AI 编程看到的一条新的路径。

这里并没有在讨论是否要按传统专业的编程思路来学习，而是提供一种更低的入门到创造一款到前后端实现日志增删改查的程度的产品。

我并不否认更高难度的产品需要学习一定的编程思想，我并不觉得学习代码进行确定性以及掌控能力更强的编程是一种对独立开发者高效的路径。

反而我们要尽量避免为了使用工具而学习工具，如果能直接达成目标，是否可以简化工具或者不需要工具是我们作为创造者所追求的，即目标导向。

我们的目标是服务用户，代码或者编程是到达目标的桥梁，如果有更快的路径，选择它就好，如果有些路还是必须要走，那么等到时候再走也来得及，重要的是快速行动，快速触达用户，快速完成价值交换与获得反馈。

希望这是一个非零和的讨论，写了不少，有感而发，我也很高兴越来越多人能参与到推进国内 AI 编程生态的完善😊

注：此处泛互联网人群指代非研发的互联网研发流程上的人群（如产品经理、运营、测试、运维等）以及白领人群（如运营小编、文员等）

> **被转动态**
> 原作者：@空格_
> 发布时间：5/6/2025, 7:09:30 PM
> 圈子：AI探索站
>
> 很多在教授 Vibe Coding 的课程多围绕传统软件开发的技术栈——前端、后端、数据库，通过实践教学制作网站、APP和小程序。
> 
> 他们试图传达这样一个观点：有了 AI，人人都可以成为全栈开发者。但现阶段 Vibe Coding 仍处于发展初期，它对软件开发的影响尚未清晰。
> 
> 对一个不懂编程的小白用户，仅依靠与 AI 对话，是无法完成一个包含前端、数据库和后端服务的复杂产品的。
> 
> 没有扎实的功底，仅靠AI很难驾驭复杂项目的开发和维护。
> 
> 面向初学者的 AI 编程课程应该明确指出：目前 AI 的能力仅限于帮助开发静态网页，工具型的小程序或APP，比如博客网站、计算器、笔记、待办清单等。
> 
> 不过仅这些，也足够发挥个人的创造能力了。和创造能力最相关的技能的是设计和产品思维，而非代码能力，这也是很多课程忽略的重点。
> 
> 所以，我想分几篇文章或视频来介绍下 Vibe coding 的入门，从一个产品经理的角度切入，如何让个人掌握 vibe coding 必备的产品、设计和开发的基础能力。本篇介绍下工程能力——Coding 必备的周边技能.
