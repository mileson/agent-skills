从大厂设计师到超级一人公司：6000字回顾我和AI的2025
==============================

原创 歸藏的 AI 工具箱 歸藏的 AI 工具箱 [歸藏的AI工具箱](javascript:void\(0\);)

其实本来是没有写年终总结的习惯的。想着好不容易到了年底，顺便又是圣诞和元旦，没有什么要发布的东西，可以休息一下。

  

但是今天早上看到 Manus 被收购的消息，然后又看到了智谱的 IPO，感觉今年确实发生了很多事情。顺便昨天看到乔木写的总结，我觉得也挺有意义的，还是写一下吧。

  

今年确实做了很多事情，生活和 AI 领域都发生了很大变化，所以还是决定写一下。

  

过去一年，我身份上的变化主要是从大厂的设计师变成了一个自由职业者（灵活就业者）。

也终于不再用数据审判自己，而是把注意力放回“我擅长什么、我愿意长期做什么”：在保持松弛的同时提升节奏，用 AI 加持把歸藏做成一个能持续输出、能带来收入、也能帮助行业朋友的超级一人公司。

  

 AI 自媒体
-------

先来看一下今年自媒体的成绩单。

  

我一直不觉得自己是一个自媒体，也不把自己当做一个自媒体，甚至不是一个很称职的自媒体。我很少关注大号，很少看别人的数据，也很少看自己的数据。

  

这件事有好有坏：

1.  1.好的方面：我不容易内耗，不容易产生数据焦虑
2.  2.不好的方面：有时候因为不看数据，反而会被数据“搞”。比如算法的调整、整个生态的变化，这些都需要去适应，而我适应得往往比别人慢一些

  

年初时，其实我对于这块没有什么明确的目标，也没有什么方法论。

今年这一块虽然说没有刻意去总结，但成长确实比较多。

因为对任何人来说，自媒体、营销、推广，或者创造自己的影响力，都是一个需要从头学习的过程：学校里是不教这些的，工作中所学的和在公域平台上的玩法也不太一样。

但是随着 AI 的发展，以及大家在 AI 加持下个人能力边际的提升，这部分能力应该会变得越来越重要。

  

Twitter 和即刻一直是我的主阵地。我不把即刻当作一个媒体渠道，而更像是一个大号的朋友圈。在这里跟大家交朋友，也确实认识了非常多的朋友。

  

所以说在即刻上我没有什么功利心：有什么消息就跟大家同步一下，刷到朋友有什么好玩的就转发一下。即便如此，我也依然达到了快 25,000 的粉丝量，这在即刻里面应该算是相当高的了

  

  

  

关于 Twitter，它今年的变化是非常大的。

  

老马在成立 xAI 并迅速训练出 Grok 模型以后，对 Twitter 的算法进行了大幅度的优化，整个生态也发生了巨大的变化。总的来说，虽然过程有些曲折，但我觉得这些优化和算法的变化是相对有益且成功的。

  

我也经常说，如果你是做 AI 的，却不刷 Twitter、不用 Twitter，那你跟这个行业是有隔阂的，与行业最前沿的信息也会存在差距。

  

Twitter 我是玩得比较舒服的，对于各种东西的变化也比较敏感。我也很高兴有很多 AI 圈的朋友们开始玩 Twitter，开始跟海外的一些朋友互动。

  

这也是我觉得大家玩 Twitter 应该注意的一个地方：

尽量的，即使你在用中文，也应该尽量的去跟海外的用户、那些前沿的公司负责人和研究员多进行互动。你既然都发了，来都来了，对吧？尽量要让自己融入到这个环境和体系里，不要老在简中圈“圈地自萌”，只在意数据。

  

Twitter 今年的粉丝量截止目前达到了 11 万。

  

  

  

我曾经有一段时间非常焦虑，因为怎么发都不涨，但是后面重新想了一下自己的定位以后，就好了很多，也不再纠结粉丝量和数据的问题，它反而自然而然就涨起来了。

  

除了这些，我还欠了很多企业 B 端朋友的一些 Twitter 或者海外增长课。这些内容目前也在制作中，我觉得这部分的经验对于企业或出海公司来说是非常有用的。

  

而且在整个简中圈，我做 Twitter 海外粉丝的数量应该是相对非常多的，很多人甚至会点击翻译来查看我的内容。所以这部分我也积累了一些心得，希望能分享给大家。

  

公众号是今年主要做的一个地方，这个不避讳，因为确实公众号的商业价值是最高的。

但是微信的算法确实比较蛋疼，公众号今年基本上是从零开始做的。从今年 DeepSeek 开始，过年后当时只有 1 万多的粉丝量，现在已经做到了 7.8 万左右。

而且公众号今年也做了一些爆款，有一些内容大家也非常喜欢。希望明年能有一个 10 万+ 吧。

  

  

  

自媒体上最大的一个变化是我开始做视频了。我开始尝试用一些简单、快速的方法，哄着自己把视频做出来。

  

说实话，有几个视频在一些平台的数据还是非常不错的：抖音和小红书：数据表现很好，视频号：数据也非常好，产出了很多几十万播放，甚至上百万播放的视频。

  

鉴于 AI 洗稿的专业度以及 AI 的创意文学能力越来越高，我整体对于图文内容的商业价值是持悲观态度和看法的。所以，做视频可能是一个大家不得不面对的选择。

  

  

  

关于自媒体，今年我还是相对比较满意的。

  

作为一个不把自己当做自媒体的自媒体人，能有现在这样的影响力，包括在各个平台上的粉丝量，我感到很开心。我经常开玩笑说，我一个人运营 9 个平台，还得持续更新视频，要是没有 AI 是根本行不通的。

  

经常有人问我：“AI 到底有什么用？”我的回答很简单：如果没有 AI，一个人不可能同时更新 9 个平台，对吧？

  

这部分写起来可能就比较长了。如果大家感兴趣，我找时间去演示或者写一下我是如何整理和发布内容的，以及如何高效地一个人更新各个平台的内容。

  

AI 社群
-----

  

其实我是很讨厌维护和管理社群的，因为要花太多的时间和精力。

  

但去年为了保证 AIGC 周刊的更新，不得已采取了收费策略。搞了一年以后发现也还行，大家都非常省心，社群里的人也都非常好，能够进行互助和分享。

  

  

  

我只需要稍微引导一下就行，并没有付出非常大的管理成本。

  

今年社群上一个比较大的变化是，我探索并尝试了一种新的推广方式：

在卖社群的同时，帮大家去推广一些国产的 AI 产品。我想通过自己的方式，为这些服务、为整个行业的发展尽一份力，让大家能够更多地体验到这些优秀的产品，也让他们能够多一些转化。

  

所以在今年11月初，我搞了一个比较大的活动，基本上拉上了所有我认识的朋友或者是AI产品界的朋友。

  

  

  

简单进行一下复盘：

这次会员大概卖了2000个（算上老会员的续费），而且这2000个全都是付费会员，在中国算是规模比较大的社群了。当然，这还是在我推广力度非常小的情况下实现的。

  

虽然这期间我比较累（因为要一个一个手动添加会员微信，还有一些沟通上的琐事），但让我很开心的是，我竟然坚持一个人把这件事做下来了，现在想想也觉得很离谱。

  

在这个过程中，我需要独立完成这些工作，确实很累，也出现了一些值得优化的问题：

1.  1.与30多个产品区的负责人进行沟通
2.  2.确定活动时间
3.  3.制定营销方案和优惠方案
4.  4.一个人负责所有的抽奖环节
    
      
    

但我最高兴的是，这种活动方式确实能帮助到一些 AI 创业者。很多产品负责人朋友跟我说，我开始会员优惠活动的那一天，可能是他们在“黑五”或年底大促之前，销售额最高的一天。

  

同时也促使很多朋友建立了属于自己的营销体系和营销方案，包括一些优惠方案等。在此期间，大家在参考他人的营销方案或方式时，也会不断进行优化。

  

所以很开心能在收入和营销方案上，切实地帮助到一些产品。此外，我也帮会员朋友们争取到了不少福利：

1.  1.抽奖环节持续了快20天的时间
2.  2.每天安排 3 个抽奖
3.  3.基本上是百分之百的中奖率

  

最后真的感谢小报童、Quaily，以及参与这个活动的 AI 产品的朋友和 AIGC 周刊社群的所有朋友。

  

AI 创作
-----

今年 AI 创作的主线与 AI 的发展肯定是分不开的，主要体现在以下几个方面：

  

1.  1.Vibe Coding 的兴起：由于模型代码能力的增强，Vibe Coding 开始飞速发展。我自己也尝试了许多 Vibe Coding 项目，并发布了一系列教程。
2.  2.Agent 的发展：我自己也亲手构建了不少 Agent，并向大家科普了 MCP 相关内容。这几个板块的热度都非常高。
3.  3.图像与视频领域：这部分大家应该都很熟悉了，是我涨粉最多的领域。作为一名设计师，这可能也是我的舒适区。

  

Vibe Coding 领域：这部分刚开始是关于网页生成的提示词。目前很多产品中还能看到这些提示词的影子。基本上，引用现成的前端 CDN 服务直接进行 Vibe Coding 或网页生成，这个模式就是我和乔木等其他几个人推起来的。

  

关于 Agent，目前主要是 Vibe Coding 的 AI 编程工具。除了这些 Agent 产品，我还产出了关于 Agent 周边上下文服务（如 MCP）兴起的这部分内容。

  

主要包括比如刚开始的 Gemini CLI 教程、MCP 的一些使用教程。Chatwise 、Youware、Trickle 这种非常好用的工具教程。

  

  

视频图像方面的内容就非常多了，包括：从刚开始的 ComfyUI 教程、到后来 GPT-4o 图像模型发布时，我有许多提示词作品出圈、再到后来的 Nano Banana 和 Nano Banana Pro、以及 Kontext、Seedream 4.0 这种图像编辑模型。

  

  

  

当时有非常多的提示词作品出圈。经常在极短的时间内，打开小红书搜索原文，就能看到好几篇一模一样的内容。大家总开玩笑说：我养活了半个 AI 圈。

我自己是非常愿意大家拿着玩的，因为毕竟这样东西有人用，才说明内容和提示词的价值。但最好还是自己去玩一玩、试一试，不要照着抄。如果你艾特我的话，我还能帮你点个赞。

  

我也开了一个 GitHub 的仓库，尽量把我今年觉得比较好的一些提示词整理一下放在里面。当然现在还在整理哈，我会慢慢地把它们都放进去。如果你对这些东西感兴趣，可以来这里（github.com/op7418/guizang-s-prompt），或者直接在我公众号里看。

  

  

歸藏的 AI 产品朋友们
------------

今年其实除了上面这些内容以外，我还在不断地跟一些行业里的朋友接触。

包括比较深度地参与一些产品的方向性或宣发类的工作。从他们身上我学到了非常多的东西，也保证了我自己在 AI 产品或者是行业认知上的持续更新。

  

### Medeo 

一句话介绍一下 Medeo：它是一款可以与你共同进行视频创作的 AI Agent。它支持目前市面上超多的模型，并允许用户通过自然语言直接进行视频的编辑和修改。

其实我还有一个身份是 Medeo 的顾问，跟王冠、遥行、晨然接触都非常多。

Medeo 可以说是我一直看着做到现在的，每个人都非常厉害，团队氛围也特别好。他们对于 AI 领域的理解，以及对相关产品的一些深度思考，都非常有价值。

  

### Youware

一句话介绍一下 Youware：这是一个 Vibe Coding 工具以及社区。即便你是一个小白，也可以轻松地通过自然语言创建自己的网页，并与朋友们分享，实现快速部署。同时，他们还拥有非常全面的作品发布和评价体系，可以帮助你的 Web Coding 作品被更多人看见。

跟 Youware 的团队的接触也比较多，他们的迭代效率，对于编程社区和编程工具的理解都非常的到位，抓机会的能力也很强。

  

### Listenhub

一句话介绍：除了 ElevenLabs 最专业的 AI 音频平台，现在已经进化为了一个全面的 AI 创作平台，你可以在上面进行声音克隆、AI 播客、AI PPT 和视频的生成。

橘子可以说是我在这个行业的半个老师，他在 AI 产品、营销和商业化上的很多洞察都非常厉害，而且还可以很容易的启发到你，而且也一直在实验和践行 AI 优先的团队管理理念，有了不少的成果。

  

  

当然还有很多非常优秀的从业者和公司都有很多的交流，也帮了我很多这里就不一一列举了。

  

2025 年的 AI
----------

既然是 AI 总结的话，还是要聊一下行业。虽然说大家都比较熟悉了，但还是回顾一下，微博给的这三个问题比较好，我就直接在这里顺便写了。

  

### 2025年将AI推入一个新阶段的关键技术突破是什么?

  

我觉得主要有两个方面（如果非常精简的话）：

  

首先是强化学习在工具使用以及编程领域的应用：

  

这极大地促进了 Vibe Coding 层面和 Agent 工具的爆发。去年这个时候，还有很多朋友说 AI 编程不靠谱、不能用，甚至以不用 AI 编程为荣；但到了今年，已经很少听到这种声音了。

  

甚至有很多朋友说，Claude Code 宕机的时候，他们就没办法工作了。

  

第二个就是多模态的爆发；

  

去年大家对于多模态的理解，还停留在理解图片信息、进行一些简单的图像推理层面。甚至视频的多模态理解在当时都不是一个重要的、受人关注的领域；而音频理解，在那时可能仅仅被视为简单的语音转文字。但是到了今年，大家终于看到了多模态训练在输出方面的威力，以及强大的多模态推理能力对输出辅助带来的显著提升。

  

举个简单的例子：比如在视频领域，我们的 Sora 2 和 VEO 3.1 在多模态直出和分镜编排上带来了巨大的突破。

  

与此同时，Nano Banana Pro 和 Seedream 4.0 这些模型在涉及到世界知识以及图像推理的过程中，生成的图片质量也在飞速提高。这些进展可以快速嵌入到原先的 Agent 体系中，让大语言模型真正变成它的“脑袋”

  

### 2025年有哪些AI产品让你眼前一亮?

  

年初让我印象深刻的产品，其实 Chatwise 算一个。

  

因为目前我们所有的内容或模型都分布在各个平台，虽然它们各有优势，但没有一个产品能将这些优势整合起来。

  

Chatwise 的核心优势在于：

1.  1.整合能力：将各个模型的优势集成在了一起
2.  2.工具加持：拥有各种工具的辅助，包括对 MCP 的支持
3.  3.高效构建：能让我用非常简单的提示词，就“搓”出一个所谓的 Agent

  

然后就是 Manus：关于 Manus，虽然大家对其可能有各种各样的想法，或者是不看好的部分，但它确实是除 Cursor 这种编程工具以外，第一个让大家认识到通用 Agent 应该长什么样的产品。它在 Agent 交互领域的演示方式及其构建方法上，迈出了非常多的第一步，给了整个行业巨大的启发。

  

当然最重要的还是 Claude Code。Anthropic 公司虽然它的价值观在某些层面上有些问题，但你不可否认它在推动 AI 编程落地，以及在 Claude Code 上表现出的一些先进的人机融合工作方法上，确实非常天才。

这其中包括了：之前的 MCP、后面的 Skills、内部非常多精妙的设计。

  

硬要说的话，还有一个我觉得就是 AI Studio 的 Build 模式。

  

这纯粹是谷歌的大力出奇迹，让你可以看到一个厂商在 AI 模型领域全面开花并整合到一个产品后，所诞生的威力和强大效果。尤其是在 Gemini 1.5 Pro 更新以后，这方面的优势就更为凸显。

  

如果你还没有用过 AI Studio Build 模式去创建项目或者尝试 Vibe Coding 的话，我强烈推荐你试试。它几乎完全用 AI 模型替代了所谓的后端。

  

### 展望2026哪个趋势是不可忽视的路标?

  

目前来看，一些趋势仍会继续变化：

强化学习领域的应用：Gemini 3 Pro 和 Flash 的进化已经让我们看到了一些希望。

  

算力中心的构建：随着 OpenAI 和 Stargate 在大型数据中心、算力中心的建设完成，这两家公司一定会拿出更大、更强的模型来给我们惊喜。

  

在多模态领域：视频的 Nano Banana Pro 时刻是我一直以来非常期待的。目前来看，我们在视频领域的架构迭代和更新相对偏慢。Gemini 3 Pro 很好，但它又不是那么好。我们需要的是全面的理解、编排以及编辑。

  

随着模型能力的增强以及我们 AI Coding 基建的完善，Vibe Coding 在 2026 年将会迎来它的 DeepSeek 时刻（也就是一次极大的破圈）。而且我觉得这个时刻会来得非常快，大家拭目以待。

  

在 Agent 的产品上，我觉得大家目前已经清醒了，不再寄希望于构建所谓的通用 Agent，而是开始在各个不同的垂直领域发力，目前也已经有了一些不同领域的标杆产品。

  

随着工具调用能力的增强、上下文的增强，以及上下文管理技术的突破，我觉得 2026 年应该还会有一次突破。但是具体突破的表现形式是什么，这个我暂时想不明白，或者说还没有太看清楚。

  

当然也期待可能水下还有很多我们不知道的事情正在发生，某天就会突然爆出一个大的突破。比如说过去两年，每年春节都会有大动作。今年春节不知道是哪家公司会给我们带来惊喜？
