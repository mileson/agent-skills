## 超简单配置教程

下面来和大家介绍下如何配置PagesMCP，过程超级简单：

### 第一步：配置MCP服务

打开Windsurf，点击锤子图标，点击Configure：

![Image](./assets/17516878978280.1325697439757918.png)

### 第二步：添加配置代码

在左侧文本框里最后的位置，复制粘贴进去下面这段代码：

```
"edgeone-pages-mcp-server": {  "command": "npx",  "args": ["edgeone-pages-mcp"]}
```

不用担心格式是否正确，我们全选上所有的代码，然后点击"Chat"：

![Image](./assets/17516878976900.8316765049693547.png)

然后输入："帮我检查下代码格式"

![Image](./assets/17516878976860.3386036785600749.png)

然后Windsurf会自己去修复格式，我们只需要点击右下角的"Accept all"就可以了：

![Image](./assets/17516878981700.6915894633425219.png)

这个点其实是我很喜欢用Windsurf类AI IDE的原因 —— 如果你用CherryStudio/ChatWise这类壳，是不方便做类似的AI自动修复调整的。

### 第三步：刷新MCP服务

继续点锤子图标，然后点"Refresh"：

![Image](./assets/17516878979270.5276127417246113.png)

有可能需要多刷几次，现在MCP server和Windsurf这类MCP Client之间连接还不大稳定。直到edgeone-pages-mcp-server前面显示绿灯，说明配置完成了。

搞定！接下来，你就可以愉快地白嫖腾讯的PagesMCP了！尝试在先前完成开发的本地静态网页里喊出咒语吧：

```
帮我把网页部署到线上
```

## PagesMCP的技术原理

根据腾讯云官方的技术说明，PagesMCP的实现原理相当巧妙：

1. Pages Functions集成：PagesMCP将MCP服务与EdgeOne Pages Functions（一个无服务器计算平台）深度集成，允许在边缘执行JavaScript/TypeScript代码。

2. 工作流程：

* 当你在Windsurf等编辑器中输入"帮我部署网页"类似的指令时
* AI生成HTML内容，PagesMCP服务接收这些内容
* 自动将内容部署到EdgeOne Pages平台
* 返回一个可公开访问的URL

3. 无缝体验：整个过程对用户完全透明，不需要任何额外的操作或登录步骤，真正做到了"一句话部署"。

这种设计充分体现了MCP技术与边缘计算的完美结合，为开发者提供了前所未有的便捷体验。

目前，EdgeOne Pages（包括PagesMCP）处于公测阶段，完全免费使用。从腾讯云的定价策略来看，即使未来开始收费，也很可能会保留免费额度。

对我们个人开发者和小型项目来说，即使未来收费，个人版或基础版也完全够用，不用担心成本问题。
