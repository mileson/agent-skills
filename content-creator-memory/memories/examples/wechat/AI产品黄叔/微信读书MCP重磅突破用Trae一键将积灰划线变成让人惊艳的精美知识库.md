## 安装和配置微信读书MCP

首先，我们需要解决一个小问题：如何获取微信读书的数据。考虑到Cookie经常过期的问题（这个真的很烦人！），我推荐使用CookieCloud来自动更新Cookie，这就像给你的数据通道装了一个永不断电的备用电源。

CookieCloud是一个浏览器插件，Chrome和Edge用户都可以在应用商店搜索获取：

![Image](./assets/17516879177740.40915489636733415.png)

安装完毕后，按照下面步骤进行配置（比配置路由器简单多了）：

1. 先登录微信读书网页版 https://weread.qq.com，确保生成有效Cookie

1. 1. 服务器地址：使用我自建的服务器 https://cc.chenge.ink 或填入自建服务器地址
2. 2. 点击"自动生成密码"（省去了自己想密码的麻烦！）
3. 3. 同步域名关键词中填入"weread.qq.com"
4. 4. 【可选】配置Cookie保活（建议开启，避免频繁失效的烦恼）
5. 5. 点击"保存"，然后点击"手动同步"确保配置生效

7. 完成配置后，记得复制并保存服务器地址、用户UUID、加密密码三个字段，这就像是你的数据金库的钥匙，后续配置要用到。

![Image](./assets/17516879162790.22015100474996918.png)

## 在Trae中配置微信读书MCP

原作者在blog里介绍的是Claude/Cursor的配置方法，但其实在Trae里配置更简单，就像给手机装App一样直观。我们打开Trae，依次进入：

我的 → AI功能管理 → MCP

在MCP界面里点击"添加"，然后在MCP/市场里，点击"手动配置"，填入以下代码（就像输入魔法咒语一样）：

```
{
  "mcpServers":{
    "mcp-server-weread":{
      "command":"npx",
      "args":["-y","mcp-server-weread"],
      "env":{
        "CC_URL":"https://cc.chenge.ink",
        "CC_ID":"您的CookieCloud UUID",
        "CC_PASSWORD":"您的CookieCloud密码"
      }
    }
}
}
```

然后点击"确认"，就完成配置了！在MCP界面里，当你看到mcp-server-weread后面显示✅可使用时，说明已经安装成功，简单得让人不敢相信：

![Image](./assets/17516879169640.4600928612206674.png)

## 打造你的智能读书笔记生成器

上面这一步只是搞定了微信读书的数据提取，就像我们准备好了食材。如果想一步生成精美卡片，还需要配置Trae的智能体——这就是我们的大厨了！

打开我的 → AI功能管理 → 智能体，在界面里先把右下角的"自动运行命令和MCP工具"勾选上（这样Trae就能自动帮我们处理一切），然后点击"创建智能体"：

![Image](./assets/17516879183480.913739020025255.png)

在创建界面里，需要填三个选项：

名称：

```
黄叔*向阳自由风格卡片
```

这个提示词主体要感谢@向阳乔木，我做了一些微调，最主要的是去掉了设计的高度限制，这样就能生成很长的卡片，方便AI更全面地提炼划线笔记的内容——就像把一页纸变成了可以无限延展的画卷。

提示词：

（这里是一段很长的提示词，我就不全部贴出来了，感兴趣的朋友可以关注”AI产品黄叔“公众号，转发这篇文章，然后发送截图和回复“微信读书“获取完整提示词）

工具：

记得把mcp-server-weread勾选上，这是允许智能体调用刚才我们创建好的微信读书MCP，就像给厨师准备好了食材处理机。

最后点击保存，在弹出的对话框里可以点击"立即使用"，开始调用MCP了！
