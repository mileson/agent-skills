# 功能导向 Git Commit

## 核心原则

Commit 信息应描述**用户能感知的变化**，而非技术实现细节。

| ✅ 正确示例 | ❌ 避免写法 |
|-----------|-----------|
| - 修复,相机界面底部布局偏移问题 | 优化，CameraPreviewView 添加 frame(maxWidth: .infinity) |
| - 新增,按钮禁用状态的视觉反馈 | 新增，ColorTheme.createSaveButtonDisabledBackground 配色 |
| - 精简,多语言配置以提升启动速度 | 优化，Localizable.xcstrings 从 4.1 万行减少到 8000 行 |
| - 修复,弹窗点击穿透到底层控件的问题 | 新增，TouchBlockingView.swift 基于 UIKit hitTest 机制 |
| - 修复,iOS 17 上姿势推荐面板点击失效 | 优化，姿势推荐面板添加 FullScreenTouchBlockingView 拦截层 |

## 写作规则

1. **聚焦用户价值**：描述对用户的影响
2. **使用祈使句**：新增/修复/优化/移除/改进/更新
3. **简洁明了**：标题控制在 50 字符以内
4. **避免技术细节**：不出现文件名、函数名、参数值

## 动作词选择

| 动作词 | 适用场景 |
|-------|---------|
| 新增 | 添加新功能、新组件、新配置 |
| 修复 | 修复 Bug、兼容性问题 |
| 优化 | 性能优化、体验优化、代码重构 |
| 改进 | 功能改进、体验改进 |
| 移除 | 删除冗余代码、废弃功能 |
| 更新 | 数据更新、配置更新、文档更新 |
| 调整 | 参数调整、布局调整 |
| 整合 | 功能合并、组件整合 |
| 精简 | 精简配置、减少文件体积、移除冗余数据 |

## 代码关键词 → 功能描述映射

| 代码关键词 | 功能描述 |
|-----------|---------|
| 点击穿透、hitTest、allowsHitTesting | 修复点击响应问题 |
| 布局偏移、frame、padding、spacing | 修复界面布局问题 |
| 本地化、Localization、多语言 | 更新多语言支持 |
| 弹窗、Sheet、Popup、overlay | 优化弹窗交互 |
| 配色、Color、Theme、background | 优化视觉样式 |
| 网络、API、请求、响应 | 优化网络请求 |
| 缓存、Cache、UserDefaults | 优化数据缓存 |
| 动画、Animation、Transition | 优化动画效果 |

## 执行步骤

1. 运行 `git status` 和 `git diff --staged` 查看变更
2. 提取每项变更的用户价值（用户能感知到什么变化？）
3. 使用动作词开头，聚焦问题/功能本身
4. **总结标题**：将所有条目概括为一句简短标题（不超过 20 字符）
5. **自动执行** `git commit -m "<标题>\n\n<内容>"`（无需二次确认）

## 执行约束

- **仅执行 Commit**：本 skill 只负责生成 commit 信息并执行 `git commit`，**禁止执行 `git push`**
- **自动提交**：生成 commit 信息后自动执行提交，无需询问用户确认
- **静默结束**：提交完成后直接结束，**不要询问用户是否需要 push**

## Commit 信息格式

```
<精简标题>

- 动作词,功能描述
- 动作词,功能描述
```

**格式说明**：
- 标题：一行概括，20 字符以内，聚焦核心价值
- 空行：标题与内容之间空一行
- 内容：具体条目，每行以动作词开头
- Co-Authored-By（可选）：`Co-Authored-By: Claude Opus 4.5`（不含邮箱）
