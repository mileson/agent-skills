# 阶段4: 开发执行（Greenfield）

## 目标

基于模块需求文档、模块技术文档、版本化执行跟踪文档和版本化 TODO 清单，循环完成开发、验证和文档回写。

## 开发前门禁

进入编码前必须同时满足：

1. 已读取对应模块需求文档
2. 已读取对应模块技术文档
3. 验收标准已明确
4. 如涉及高风险问题，已读取对应 `ProblemSolutions`
5. 如涉及复用能力，已读取对应 `CommonCapabilities`

## 推荐执行流程

1. 读取 `Documentation/DevLogs/v1.0_执行跟踪.md`
2. 读取当前 TODO 文件，例如 `Documentation/DevLogs/v1.0_TODO_01.md`
3. 从当前未完成的最小编号任务开始执行
4. 读取该任务对应的模块需求文档
5. 读取该任务对应的模块技术文档
6. 必要时读取：
   - `Framework/CommonCapabilities/*`
   - `Framework/ProblemSolutions/*`
7. 实现代码
8. 运行验证
9. 回写模块技术文档
10. 追加修订历史
11. 勾选当前 TODO 任务
12. 更新执行跟踪文档中的当前任务指针
13. 当前 TODO 文件完成后，再进入下一份 TODO 文件
14. 继续下一个任务

## 顺序执行规则

- 必须按任务编号顺序推进
- 完成 `TODO-001` 后，才能进入 `TODO-002`
- 不允许跳过未完成的前置任务
- 如果任务依赖未满足，必须先回到对应前置任务
- `TODO_01.md` 完成后，才允许进入 `TODO_02.md`
- Agent 必须保证任务执行链可以从 001 连续推进到后续编号

## 文档回写要求

开发完成后，至少要同步：

- 模块技术文档中的 `实施现状`
- 模块技术文档中的 `代码同步说明`
- 该文档的 `文档修订历史`
- 对应版本的 `执行跟踪.md`
- 对应版本的 `TODO_01.md / TODO_02.md ...`

如有必要，再同步：

- 模块需求文档（需求或验收发生变化时）
- `Framework/CommonCapabilities/`
- `Framework/ProblemSolutions/`

## 验证要求

验证至少覆盖：

1. 核心业务闭环
2. 关键异常路径
3. 关键状态流转
4. 文档与实现是否一致

## 完成标准

- ✅ 核心任务已完成
- ✅ 验证通过
- ✅ 模块技术文档已回写
- ✅ 修订历史已追加
- ✅ 执行跟踪文档已更新
