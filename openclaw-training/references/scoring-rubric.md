## 评分前置条件（必须）

> ⚠️ **未经教练实地验证的任务，不得进入评分环节。**

评分前，教练必须完成以下验证（详见 [training-framework.md](training-framework.md#产出物实地验证规范)）：

1. **文件类产出**：SSH 读取内容，确认非空且有实质信息
2. **图片类产出**：下载到本地查看，确认文字渲染正常、无乱码
3. **平台发布类产出**：通过工具或 URL 访问实际帖子，确认内容和图片正常
4. **代码/脚本产出**：远程执行验证，确认可运行

验证结果必须记录在 session-log 的"教练验证"区块中。

## 评分体系

每个任务满分 **100 分**，按 6 个维度加权计算：

| 维度 | 权重 | 说明 |
|------|------|------|
| 完成度 | 25% | 任务是否完成、交付物是否齐全 |
| 质量 | 25% | 产出内容的专业性、准确性、深度 |
| 工具使用 | 15% | 是否使用正确的工具、用法是否规范 |
| 规范遵守 | 15% | 是否遵循 AGENTS.md/TOOLS.md 中的规则 |
| 效率 | 10% | 完成速度、token 消耗合理性 |
| 自主性 | 10% | 是否需要额外提示、是否自主决策 |

## 六维度评分标准

### 完成度（25%）

| 分值 | 标准 |
|------|------|
| 25 | 所有交付物齐全，超额完成 |
| 20 | 所有交付物齐全 |
| 15 | 主要交付物完成，次要有缺失 |
| 10 | 部分完成，核心交付物缺失 |
| 5 | 仅启动但未完成 |
| 0 | 未执行或完全失败 |

### 质量（25%）

| 分值 | 标准 |
|------|------|
| 25 | 内容专业、结构清晰、数据准确、可直接使用 |
| 20 | 内容良好，偶有小瑕疵 |
| 15 | 内容基本合格，存在明显可改进处 |
| 10 | 内容粗糙、结构混乱 |
| 5 | 内容偏题或大量错误 |

### 工具使用（15%）

| 分值 | 标准 |
|------|------|
| 15 | 使用正确工具、参数规范、错误处理到位 |
| 10 | 使用正确工具但参数有误或缺少错误处理 |
| 5 | 使用了非指定工具或工具使用方式不当 |
| 0 | 未使用工具或使用完全错误 |

### 规范遵守（15%）

| 分值 | 标准 |
|------|------|
| 15 | 严格遵循所有规范（命名、格式、路径、铁律） |
| 10 | 基本遵循，有 1-2 处轻微违规 |
| 5 | 多处违规但未触犯铁律 |
| 0 | 违反铁律级规则（如使用禁止的工具） |

### 效率（10%）

| 分值 | 标准 |
|------|------|
| 10 | 在预期时间内高效完成 |
| 7 | 略超预期时间但完成 |
| 4 | 明显超时或多次重试 |
| 0 | 超时未完成 |

### 自主性（10%）

| 分值 | 标准 |
|------|------|
| 10 | 完全自主完成，无需额外提示 |
| 7 | 需要 1 次澄清/提示 |
| 4 | 需要 2-3 次提示 |
| 0 | 完全依赖引导 |

## 评级对照表

| 分数区间 | 评级 | 说明 |
|---------|------|------|
| 95-100 | S 卓越 | 超额完成，可作为范例 |
| 90-94 | A 优秀 | 全面达标，质量突出 |
| 85-89 | B 良好 | 基本达标，有改进空间 |
| 80-84 | C 合格 | 勉强达标，需要关注 |
| 70-79 | D 不足 | 多处问题，需针对性强化 |
| <70 | F 不合格 | 未达到基本要求，需重训 |

## 扣分规则

以下情况额外扣分（在维度得分基础上）：

| 违规类型 | 扣分 |
|---------|------|
| 违反铁律（如用浏览器代替指定工具） | -20 |
| 敏感词泄漏（内部称呼出现在外部文件） | -10/次 |
| 产出文件为空或损坏 | -15 |
| Session 膨胀导致后续任务失败 | -5 |
| 产出物存在可见缺陷但 Agent 未自行察觉（如图片乱码、格式错误） | -10 |

### 教练自律规则

| 教练违规类型 | 处理方式 |
|------------|---------|
| 未实地验证就评分 | 发现后必须重新验证并修正评分 |
| 仅凭 Agent 文字报告评分 | 同上，且在报告中记录"教练反思" |
