# 阶段1: 需求沟通（Greenfield）

## 目标

通过多维度、多层次的提问，充分理解用户需求，生成详细的 PRD（产品需求文档）。

## 多维度提问框架

### 维度1: 产品概述

**目标**: 理解产品核心定位和价值主张

**关键问题**:
1. 这个产品/系统的核心价值是什么？解决什么问题？
2. 目标用户是谁？他们的痛点是什么？
3. 与竞品相比，有什么独特优势？
4. 产品的核心功能是什么？（列出 Top 3-5）
5. 预期的用户规模和增长目标是什么？

**输出**: `doc/01_PRD/1_产品概述.md`

### 维度2: 用户画像

**目标**: 深入理解目标用户群体

**关键问题**:
1. 主要用户群体有哪些？（如个人用户、企业用户、管理员）
2. 每个用户群体的特征是什么？（年龄、职业、技能水平）
3. 用户使用场景是什么？（办公室、移动端、特定时间）
4. 用户的技术水平如何？（小白、中级、专家）
5. 用户期望的使用频率是什么？（每天、每周、偶尔）

**输出**: `doc/01_PRD/2_用户画像.md`

### 维度3: 核心流程

**目标**: 梳理产品的核心业务流程

**关键问题**:
1. 用户第一次使用的流程是什么？（注册、登录、引导）
2. 核心功能的操作流程是什么？（详细步骤）
3. 是否有审批流程？（如需要管理员审核）
4. 是否有支付流程？（如电商、会员订阅）
5. 是否有通知流程？（邮件、短信、推送）

**输出**: `doc/01_PRD/3_核心流程.md`（包含 Mermaid 流程图）

### 维度4: 边界场景

**目标**: 发现边界情况和异常处理

**关键问题**:
1. 用户未登录时，能访问哪些功能？
2. 用户权限不足时，如何处理？
3. 数据量很大时（如千万级用户），如何处理？
4. 并发量很大时（如秒杀场景），如何保证稳定性？
5. 第三方服务（如支付、短信）故障时，如何降级？
6. 用户误操作时（如误删数据），能否恢复？

**输出**: `doc/01_PRD/4_边界场景.md`

### 维度5: 商业模式

**目标**: 理解产品的盈利模式和成本结构

**关键问题**:
1. 产品如何盈利？（免费、订阅、广告、佣金）
2. 定价策略是什么？（固定价格、分级定价、按量计费）
3. 是否有免费试用期？试用期多长？
4. 是否有会员体系？会员权益是什么？
5. 预期的成本结构是什么？（服务器、带宽、人力）

**输出**: `doc/01_PRD/5_商业模式.md`

### 维度6: 运营模式

**目标**: 理解产品的运营策略和增长手段

**关键问题**:
1. 如何获取第一批用户？（地推、广告、口碑）
2. 如何激活用户？（引导任务、新手奖励）
3. 如何提高留存率？（会员特权、积分体系）
4. 如何促进用户分享？（邀请奖励、社交裂变）
5. 是否需要运营后台？后台功能有哪些？

**输出**: `doc/01_PRD/6_运营模式.md`

## 提问策略

### 循序渐进

不要一次性问所有问题，每次提问 3-5 个关键问题，等待用户回答后再继续。

### 发散思考

根据用户回答，继续追问细节，发现隐藏需求。

**示例**:
```
用户: "我想做一个电商平台"
Agent: "电商平台是一个很大的概念，我们先从核心功能开始：
1. 主要卖什么类型的商品？（实物商品、虚拟商品、服务）
2. 商家模式是什么？（自营、第三方入驻、混合）
3. 目标用户是谁？（C端消费者、B端企业）
4. 核心竞争力是什么？（价格优势、供应链、用户体验）"
```

### 确认理解

每个维度完成后，总结用户需求，确认理解无误。

**示例**:
```
Agent: "根据您的描述，我理解的产品概述是：
- 核心价值：为中小企业提供低成本、易用的进销存管理系统
- 目标用户：50-200人规模的制造企业
- 核心功能：库存管理、订单管理、财务对账、数据报表
- 独特优势：移动端操作、自动化对账、AI 预测补货

请问我理解得对吗？"
```

## 生成 PRD 文档

### 文档结构

每个维度生成独立的 Markdown 文档，包含：

1. **文档头部**:
   ```markdown
   # [维度名称]
   
   **文档版本**: v1.0
   **编写日期**: 2026-02-04
   **编写方式**: AI Agent 生成
   ```

2. **文档内容**:
   - 使用清晰的章节结构
   - 使用 Mermaid 流程图（核心流程）
   - 使用表格（用户画像、功能清单）
   - 使用列表（边界场景）

3. **文档尾部**:
   ```markdown
   ## 遗留问题
   
   - [ ] 待确认：支付方式是否包含微信支付？
   - [ ] 待确认：是否需要多语言支持？
   ```

### 文档审查

每个维度的文档生成后，必须向用户展示并等待确认：

```
Agent: "我已经生成了【产品概述】文档，请审查：

[展示文档内容]

请问：
1. 是否准确理解了您的需求？
2. 是否有需要补充的内容？
3. 是否有需要修改的地方？"
```

## 完成标准

所有 6 个维度的 PRD 文档生成并通过用户审查后，阶段1 完成，进入阶段2。
