# 阶段1: 增量需求沟通（Brownfield）

## 目标

基于现有 PRD，聚焦增量需求，生成增量 PRD v2.x，更新文档修订历史。

## 增量提问框架

### 与模式1的区别

- **模式1**: 全面提问 6 个维度，生成完整 PRD
- **模式2**: 聚焦增量需求，避免重复现有内容

### 提问策略

**第一轮：理解现有系统**
1. 我已读取了现有的 PRD（v1.5），当前系统的核心功能是 [总结]，是否准确？
2. 当前技术栈是 [技术栈]，是否有需要升级的地方？

**第二轮：发散需求探索**
3. 本次开发的核心目标是什么？是新增功能、性能优化、还是重构？
4. 新增功能与现有功能的关系是什么？是独立模块还是集成到现有模块？
5. 是否需要修改现有的数据模型或 API？

**第三轮：边界和风险**
6. 变更是否会影响现有功能？有哪些模块受影响？
7. 是否需要数据迁移？迁移脚本如何设计？
8. 哪些现有功能需要回归测试？

## 生成增量 PRD

### 文档版本继承

**命名规则**:
- 原文档：`1_产品概述.md` (v1.5)
- 增量文档：`1_产品概述_v2.0.md` (v2.0)

### 文档头部

```markdown
# 产品概述 v2.0

---

**文档版本**：v2.0  
**编写日期**：2026-02-04  
**基于版本**：v1.5  
**编写方式**：AI Agent 生成

---

## 文档修订历史

| 版本 | 日期 | 修订人 | 修订内容 |
|------|------|--------|----------|
| v1.0 | 2025-10-10 | 产品团队 | 初始版本：基础电商功能 |
| v1.5 | 2025-12-15 | 产品团队 | 新增会员体系、营销工具 |
| **v2.0** | **2026-02-04** | **AI Agent** | **新增智能推荐系统、用户画像** |

---

## v2.0 变更摘要

### 新增内容
- 智能推荐系统：基于用户行为数据推荐商品
- 用户画像：收集用户偏好，构建个性化标签

### 修改内容
- 商品详情页：新增"推荐商品"模块

### 删除内容
- 无

---

## 变更影响分析

| 影响范围 | 影响程度 | 说明 |
|---------|---------|------|
| 商品模块 | 中 | 需要新增推荐分字段 |
| 用户模块 | 低 | 新增画像表，不影响现有逻辑 |
| 订单模块 | 无 | 不影响 |

---
```

### 文档内容

**只编写新增/变更部分**，不重复现有内容：

```markdown
## v2.0 新增功能

### 智能推荐系统

**核心价值**：
- 提升用户购买转化率 20%+
- 提高用户停留时长 30%+

**推荐算法**：
- 协同过滤（Collaborative Filtering）
- 内容推荐（Content-based Recommendation）
- 混合推荐（Hybrid Recommendation）

**推荐场景**：
- 首页推荐：基于用户历史浏览和购买记录
- 商品详情页推荐：基于商品相似度
- 购物车推荐：基于关联商品分析

### 用户画像

**核心价值**：
- 精准营销，降低广告成本
- 个性化推荐，提升用户体验

**画像维度**：
- 基础信息：年龄、性别、地域
- 行为偏好：浏览偏好、购买偏好、价格敏感度
- 消费能力：月消费金额、客单价、购买频次

### 与现有系统的关系

**复用现有模块**：
- 用户系统：复用用户ID、登录状态
- 商品系统：复用商品数据、分类数据
- 订单系统：复用订单数据、购买记录

**新增模块**：
- 推荐引擎模块（独立微服务）
- 用户画像服务（与用户模块解耦）

**集成方式**：
- 推荐结果通过 API 返回给前端
- 前端在商品列表页、详情页嵌入推荐位
```
