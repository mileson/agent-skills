# 需求计划文档生成器

## 核心规范（必须遵循）

1. **禁止写入实现代码**：需求文档关注产品逻辑、架构、流程、说明，不写具体代码实现
2. **Mermaid 样式规范**：所有流程图必须定义 `classDef`，时序图必须包含样式头
3. **表格格式规范**：表格行之间严禁空行
4. **按需生成章节**：根据需求范围（前端/后端/全栈）只生成相关技术方案章节

## 使用流程

1. **识别需求范围**：判断是【纯前端】、【纯后端】还是【全栈】开发
2. **收集必要信息**：功能目标、用户故事、涉及的页面/组件、数据模型和接口、技术栈
3. **生成文档**：使用 `template.md` 模板结构生成文档
4. **输出文件**：将内容写入指定的 `.plan.md` 或 `.md` 文件
