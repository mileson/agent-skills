> 目标：生成有故事感、可发布的图文正文，而不是逐图说明式 PPT 文案。

- 平台：`xhs / instagram / douyin / kuaishou / wechat_channels`
- 路由：`visual_first`
- 风格：`technical-expert / storytelling / sharp-humor`

## 正文草稿

### 开头
**标题思路**：{短句、停留感、反差}

{这里写开场文案。要求：
- 先抓住用户，不要解释所有页面
- 不要出现“图1”“第一页”这类说法
- 可以直接用一个判断、一个故事瞬间，或一个反差句开场}

### 主体推进
**标题思路**：{故事感、结果推进、自然承接}

{这里写主体文案。要求：
- 文案和图序呼应，但不机械逐图讲解
- 允许一个自然段同时承接 1~2 张图的情绪或信息
- 重点是把读者一路带下去，而不是做 PPT 说明}

### 收束
**标题思路**：{总结、可收藏、可转发}

{这里写收束文案。要求：
- 给读者一个明确可带走的判断或感受
- 可以自然引出收藏 / 评论 / 继续交流
- 不要写成“图5总结如下”这种说明式结尾}

## 硬规则
- `03_image_plan.[platform].md` 仍然是图片主文件
- 本文档是图文正文草稿，不是逐图解说词
- 图片只需要与文案呼应，不要求 100% 一图一段、一页一结论
- 禁止出现 `图1 / 图2 / 第1页 / 第2页` 这类机械说明
- 禁止写成 PPT、汇报稿、操作手册口吻
