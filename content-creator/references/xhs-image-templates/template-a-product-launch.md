## 逐图详细规范

### 图 1 — 封面钩子图（必需）🎨

**目的**: 在信息流中停住用户滑动，3 秒内传达"这和我有关"+ 产品名

#### 内容定义

| 要素 | 说明 | 示例 |
|:-----|:-----|:-----|
| 痛点文案 | 不超过 8 个字，直击目标用户痛苦 | "男朋友不会拍照？" / "手机存满参考图？" |
| 产品名称 | 产品名 + emoji | 「芝士相机」📸 / 「小圆角」✨ |
| 核心卖点 | 一句话解决方案，不超过 12 字 | "一键跟拍出大片" / "免费圆角神器" |
| 视觉锚点 | 产品截图缩略 或 场景化插图 | App 界面小图 / 使用场景插画 |

#### 设计规范

| 属性 | 规范 |
|:-----|:-----|
| 尺寸 | 1080×1440（3:4 竖屏） |
| 构图 | 上 1/3 痛点文案 → 中间产品视觉 → 下 1/3 卖点+产品名 |
| 背景色 | 纯色或轻渐变（避免杂乱背景抢视线） |
| 主标题字号 | 56-72px，加粗，居中或左对齐 |
| 副标题字号 | 28-36px，Regular |
| 配色方案 | 根据产品调性选择：科技类用深蓝渐变（`#1A1A2E→#16213E`）+ 亮色点缀（`#6C63FF`）；女性向用暖粉底（`#FFF0F5`）+ 紫粉点缀（`#FF69B4`） |
| 文字颜色 | 深色底用白字（`#FFFFFF`），浅色底用深灰字（`#333333`） |
| 装饰元素 | 1-2 个 emoji 作为视觉锚点，箭头引导视线 |

#### 图片来源：AI 生图

**中文提示词**:

```
设计一张小红书封面图，竖屏3:4比例（1080×1440像素）。

画面布局：
- 顶部区域（占1/3）：大字标题"{痛点文案}"，字体加粗，{字色}，字号72px
- 中间区域（占1/3）：{产品名}的App界面截图，带手机边框mockup效果，微微倾斜15度，带投影
- 底部区域（占1/3）：副标题"{核心卖点}"，旁边放{产品名}logo

背景：{配色方案描述，如"深蓝到藏青的纵向渐变，左上角有一个模糊的圆形光晕（浅紫色）"}
整体风格：简洁现代，科技感，适合小红书女性用户审美
禁止：不要出现真实人物面部，不要出现其他App水印
```

**English Prompt**:

```
Design a Xiaohongshu (Little Red Book) cover image, vertical 3:4 ratio (1080x1440px).

Layout:
- Top third: Bold headline "{pain_point_text}" in {text_color}, 72px font weight bold
- Middle third: {product_name} app interface screenshot in phone mockup frame, slightly tilted 15 degrees with drop shadow
- Bottom third: Subtitle "{core_value_proposition}" with {product_name} logo beside it

Background: {color_scheme_description, e.g. "vertical gradient from dark navy #1A1A2E to deep blue #16213E, with a soft purple (#6C63FF) circular glow in top-left corner"}
Style: Clean, modern, tech-savvy, appealing to young female users on Xiaohongshu
Avoid: No real human faces, no watermarks from other apps
```

### 图 2 — 产品全貌展示（必需）📱

**目的**: 让用户看到产品整体质感，建立品质第一印象

#### 内容定义

| 要素 | 说明 |
|:-----|:-----|
| 展示内容 | App 主界面完整截图（最具代表性的页面） |
| 标注重点 | 用箭头/圆圈标注 2-3 个关键区域 |
| 辅助文字 | 每个标注旁加 1 行说明（不超过 6 字） |

#### 设计规范

| 属性 | 规范 |
|:-----|:-----|
| 尺寸 | 1080×1440（3:4 竖屏） |
| 构图 | 手机 mockup 居中，占画面 60-70%，周围留白 |
| 背景色 | 纯浅色（`#F8F9FA` 或 `#FFF5EE`） |
| 标注样式 | 圆角矩形气泡 + 引线箭头，气泡底色 `#6C63FF`（20%透明度），文字 `#333` |
| 标注数量 | 2-3 个，不超过 4 个（避免杂乱） |

#### 图片来源：App 截图 + 标注

**截图指引**:

```
1. 打开 App 至主界面（最完整展示功能布局的页面）
2. 使用 iOS 截图（电源+音量上）
3. 后期处理：
   - 套上手机 mockup（推荐 iPhone 15 Pro 白色边框）
   - 添加 2-3 个标注气泡：
     - 标注A → 指向最核心的功能入口，文字："一键XX"
     - 标注B → 指向特色功能区域，文字："AI智能XX"
     - 标注C → 指向操作按钮，文字："简单好用"
   - 背景设为纯浅色
```

**如果需要 AI 生成 mockup 效果的提示词**:

```
将以下App截图放入iPhone 15 Pro白色边框的手机mockup中，手机居中放置在浅灰色背景（#F8F9FA）上，手机占画面65%。在手机截图上添加3个标注气泡：
- 左上方气泡指向{功能区域A}，文字"{标注A内容}"
- 右侧气泡指向{功能区域B}，文字"{标注B内容}"  
- 底部气泡指向{操作按钮}，文字"{标注C内容}"
气泡样式：圆角矩形，浅紫色半透明底（#6C63FF 20%），深灰文字，带引线箭头。
整体竖屏3:4比例（1080×1440像素），简洁干净。
```

### 图 3 — 核心功能 A 展示（必需）📱

**目的**: 展示产品最打动人的第一个核心功能，回答"它能帮我做什么"

#### 内容定义

| 要素 | 说明 |
|:-----|:-----|
| 展示内容 | 该功能的操作过程截图（展示功能正在被使用的状态） |
| 功能名称 | 图片顶部加粗标题，如 "1️⃣ 实时姿势跟拍" |
| 操作标注 | 红框/箭头标注关键操作区域 |
| 效果说明 | 底部 1 行文字说明使用效果 |

#### 设计规范

| 属性 | 规范 |
|:-----|:-----|
| 尺寸 | 1080×1440（3:4 竖屏） |
| 构图 | 顶部功能名称 → 中间操作截图 → 底部效果说明 |
| 功能标题 | 带序号 emoji（1️⃣），32-40px，加粗，居左 |
| 截图区域 | 占画面 55-65%，可裁剪聚焦到功能区域 |
| 标注颜色 | 红色高亮框（`#FF4757`，2px 描边）或箭头 |
| 效果说明 | 底部文字 24px，灰色（`#666`） |

#### 图片来源：App 截图 + 标注

**截图指引**:

```
1. 打开该功能并操作到最佳展示状态
2. 截图后裁剪，聚焦功能核心区域
3. 后期处理：
   - 顶部加功能名称标题栏（白底 + 加粗文字）
   - 用红色矩形框高亮核心操作区域
   - 添加 1-2 个箭头指示操作方向
   - 底部加 1 行效果说明文字
```

### 图 5 — 效果展示/对比图（可选）🧩

**目的**: 用结果说服用户，前后对比增强可信度

#### 内容定义

| 要素 | 说明 |
|:-----|:-----|
| 左侧/上方 | "使用前" 状态：痛点场景或原始效果 |
| 右侧/下方 | "使用后" 状态：解决后的效果展示 |
| 分割标注 | 中间分割线 + "Before → After" 或 "使用前 → 使用后" |

#### 设计规范

| 属性 | 规范 |
|:-----|:-----|
| 尺寸 | 1080×1440（3:4 竖屏） |
| 构图 | 左右对半分割（各 540px 宽）或上下对半分割（各 720px 高） |
| 分割线 | 2px 白色线 + 中间箭头图标 |
| 左侧标签 | 红色底标签 "Before"（`#FF4757`），左上角 |
| 右侧标签 | 绿色底标签 "After"（`#2ED573`），右上角 |
| 反差要求 | 对比越明显效果越好，"使用前"可适当降低饱和度 |

#### 图片来源：拼图组合

**拼图指引**:

```
制作一张左右对比拼图：
- 左半：{使用前的场景/截图}，左上角加红色标签"Before"
- 右半：{使用后的场景/截图}，右上角加绿色标签"After"  
- 中间：白色分割线 + 向右箭头
- 整体竖屏3:4比例
```

**AI 生图提示词（当没有实际截图时）**:

```
设计一张对比图，竖屏3:4比例（1080×1440像素），左右对半分割。

左半（使用前）：
- {描述痛点场景，如"一个女生看着手机里杂乱的姿势参考图，表情困扰"}
- 整体色调偏灰暗，饱和度降低
- 左上角红色标签"Before"

右半（使用后）：
- {描述解决后场景，如"同一个女生开心地对着手机拍照，屏幕上显示姿势轮廓引导"}
- 整体色调明亮温暖
- 右上角绿色标签"After"

中间白色分割线，带向右箭头图标。
风格：小红书风格，插画或扁平设计，不要真实人物照片。
```

## 使用说明

### Agent 生成 image_plan.md 时的流程

```
1. 读取本模板的图片序列定义
2. 根据实际文章内容，填充每张图的具体内容：
   - 替换 {痛点文案}、{产品名}、{核心卖点} 等占位符
   - 根据实际功能选择核心功能A/B的展示内容
   - 判断每张图的来源类型（截图/AI生图/拼图）
3. 为每张图生成完整的：
   - 内容描述（具体到展示什么界面、什么状态）
   - 设计规范（已从模板继承，可微调）
   - 图片来源指引（截图指引 或 AI生图提示词，已填充实际内容）
4. 输出到 Output/xhs/image_plan.md
```
