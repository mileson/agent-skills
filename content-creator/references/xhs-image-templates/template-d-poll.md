## 逐图详细规范

### 图 1 — 投票封面（必需）🎨

**目的**: 一眼看懂"这是个投票/选择"，激发参与欲望

#### 内容定义

| 要素 | 说明 | 示例 |
|:-----|:-----|:-----|
| 投票主题 | 引发好奇的问题 | "选哪个？帮我做个决定！" |
| 选项预览 | A/B 两个选项的缩略预览 | 左边方案A缩图 vs 右边方案B缩图 |
| 投票引导 | 引导评论 | "评论区投票 👇" / "选1还是选2？" |

#### 设计规范

| 属性 | 规范 |
|:-----|:-----|
| 尺寸 | 1080×1440（3:4 竖屏） |
| 构图 | 上部投票主题 → 中间 A/B 缩略对比 → 下部投票引导 |
| VS 分割 | 中间放大的 "VS" 字样或 "🆚"，48px，红色 |
| 选项标签 | 左侧 "方案A"（蓝色底），右侧 "方案B"（橙色底） |
| 底部引导 | "你选哪个？评论区告诉我 💬"，28px |

#### AI 生图提示词

```
设计一张投票对比封面图，竖屏3:4比例（1080×1440像素）。

画面布局：
- 顶部：大字标题"{投票主题}"，44px加粗
- 中间（占60%）：左右对比布局
  - 左半：{方案A}的缩略效果图，上方蓝色标签"方案A"
  - 中间：大号"VS"字样（红色，60px加粗）或🆚emoji
  - 右半：{方案B}的缩略效果图，上方橙色标签"方案B"
- 底部："你选哪个？评论区投票 💬"，28px

背景：浅灰（#F5F5F5）
风格：对比鲜明，选择感强烈，让人忍不住要表态
```

| 截图 | 完整展示，不裁剪，让用户看到全貌 |
| 底部说明 | "{方案A的核心特点}"，24px |

### 图 4 — 背景说明（可选）🎨

**目的**: 说明为什么做这个投票，展示开发者诚意

#### 内容定义

| 要素 | 说明 |
|:-----|:-----|
| 背景故事 | 为什么做这个功能 |
| 用户反馈 | 引用真实用户反馈（如果有） |
| 其他方案 | 还有哪些备选（如果适用） |

#### AI 生图提示词

```
设计一张背景说明信息图，竖屏3:4比例（1080×1440像素）。

画面布局：
- 顶部：标题"📖 为什么做这个功能？"，36px加粗
- 中间：2-3个引用卡片，展示用户反馈
  卡片样式：浅色圆角矩形，左侧有引号图标
  - "{用户反馈1}"
  - "{用户反馈2}"
- 下方：开发者说明
  "收到大家的反馈后，我设计了两套方案，希望能帮到你们 🙏"
- 底部："如果都不满意，也欢迎留言提建议 💬"

背景：纯白
风格：真诚、透明、社区感
```
