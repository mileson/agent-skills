# 模板B：功能教程型

> **版本**: 1.0 | **更新日期**: 2026-02-15
>
> **适用场景**: 某个功能的使用教程、分步骤操作指南、"手把手教你"类内容

> **建议图片数**: 5-7 张（最少 5 张，最多 7 张）

> **历史案例**: 参考「做GIF原来这么简单！免费小程序分享」的教程式结构

## 逐图详细规范

### 图 1 — 结果先行封面（必需）🎨

**目的**: 用最终成果吸引用户，让人想知道"怎么做到的"

#### 内容定义

| 要素 | 说明 | 示例 |
|:-----|:-----|:-----|
| 成果展示 | 最终效果图/成品截图 | 制作好的 GIF 表情包 / 处理好的圆角图片 |
| 步骤承诺 | "X 步搞定" / "手把手教你" | "3步做出表情包" / "2分钟学会" |
| 难度标签 | 强调零门槛 | "小白也能学会" / "0基础教程" |

#### 设计规范

| 属性 | 规范 |
|:-----|:-----|
| 尺寸 | 1080×1440（3:4 竖屏） |
| 构图 | 上部成果展示（占 60%）→ 下部文字信息（占 40%） |
| 成果展示 | 产品最终效果图，高质量、色彩鲜明 |
| 主标题 | "{X}步搞定{目标}"，48-60px 加粗 |
| 副标题 | "零基础也能学会 👇"，28px |
| 背景 | 浅色底（`#F0F4FF`）或与产品调性一致的渐变 |

#### AI 生图提示词

```
设计一张小红书教程封面图，竖屏3:4比例（1080×1440像素）。

画面布局：
- 上方60%：展示{最终成果}的效果图，高质量清晰
  - {具体描述成果的视觉呈现，如"一组精美的GIF表情包排列在浅色背景上，3行3列，每个GIF有不同的表情动作"}
- 下方40%：白色/浅色区域
  - 主标题："{X}步搞定{目标}"，48px加粗，深色文字
  - 副标题："零基础也能学会 👇"，28px，灰色
  - 左下角小标签："免费教程"，圆角矩形，{品牌色}底白字

背景：{浅色渐变}
风格：干净清爽，教程感强，让人觉得"这个我也能做到"
```

### 图 2 — 步骤①（必需）📱

**目的**: 教第一个关键步骤，降低上手门槛

#### 内容定义

| 要素 | 说明 |
|:-----|:-----|
| 步骤编号 | "Step 1" 或 "① 第一步" |
| 步骤标题 | 动词开头，如"打开XX" / "选择XX" / "点击XX" |
| 操作截图 | 该步骤的 App 操作界面截图 |
| 操作标注 | 红色圆圈/箭头标注要点击/操作的位置 |
| 简短说明 | 1 行文字说明操作要点 |

#### 设计规范

| 属性 | 规范 |
|:-----|:-----|
| 尺寸 | 1080×1440（3:4 竖屏） |
| 构图 | 顶部步骤标题栏（80px 高）→ 中间截图（占 70%）→ 底部说明栏 |
| 步骤标题栏 | {品牌色}底色，白字 "Step 1：{步骤标题}"，32px 加粗 |
| 截图区域 | 裁剪到功能相关区域，不需要完整状态栏 |
| 操作标注 | 红色圆圈（`#FF4757`，3px 描边，无填充）标注操作位置 |
| 箭头 | 红色箭头指向操作目标，线宽 3px |
| 序号标记 | 如有多个操作，用 ❶❷❸ 标注顺序 |
| 底部说明 | "💡 {操作要点/小贴士}"，24px，灰色 |

#### 截图指引

```
1. 打开 App，导航到第一步的操作界面
2. 确保界面处于"未操作"状态（让用户看到起始状态）
3. 截图后裁剪到相关区域
4. 添加标注：
   - 红色圆圈围住需要点击的按钮/区域
   - 红色箭头指示操作方向（如"从左滑到右"）
   - 如有多步操作，用 ❶❷ 编号标注顺序
5. 顶部加步骤标题栏，底部加说明文字
```

### 图 3 — 步骤②（必需）📱

**目的**: 教第二个关键步骤

结构与图 2 完全一致，仅内容更换为第二步。步骤标题栏改为 "Step 2：{步骤标题}"。

**关键原则**: 步骤间保持视觉风格完全统一（同色系标题栏、同样式标注），让用户感受到连贯的教程体验。

---

### 图 4 — 步骤③（必需）📱

**目的**: 教第三个关键步骤（通常是最后一步或产出步骤）

结构与图 2/3 一致。步骤标题栏改为 "Step 3：{步骤标题}"。

**特殊处理**: 如果这是最后一步，可在底部说明栏写 "✅ 完成！效果见下一张图"。

---

### 图 5 — 完成效果展示（必需）📱/🧩

**目的**: 展示实际操作完成的效果，强化"你也可以"的成就感

#### 内容定义

| 要素 | 说明 |
|:-----|:-----|
| 效果展示 | 实际操作产出的成品截图/效果图 |
| 成就标签 | "✅ 完成效果" 或 "🎉 就是这么简单！" |
| 强调点 | 标注成品中最亮眼的部分 |

#### 设计规范

| 属性 | 规范 |
|:-----|:-----|
| 尺寸 | 1080×1440（3:4 竖屏） |
| 构图 | 成品图居中展示，周围适当留白 |
| 成就标签 | 顶部居中，绿色系（`#2ED573`），加粗 |
| 效果图 | 高质量展示，可适当美化（提高对比度/饱和度） |
| 底部文字 | "是不是很简单？试试看吧 👇"，引导用户动手 |

### 图 6 — 进阶技巧（可选）📱

**目的**: 增加笔记的"收藏价值"，提升保存率

#### 内容定义

| 要素 | 说明 |
|:-----|:-----|
| 进阶技巧 | 1-3 个高级用法或隐藏技巧 |
| 其他场景 | 展示同一功能在不同场景的应用 |
| 标题 | "💡 进阶技巧" 或 "🔥 高手都在用的隐藏功能" |

#### 设计规范

| 属性 | 规范 |
|:-----|:-----|
| 构图 | 列表式布局，每个技巧一行（emoji + 文字 + 配图） |
| 标题 | 32px 加粗，顶部 |
| 技巧列表 | 28px，行间距 1.5 倍，每条前加 emoji 序号 |

---

### 图 7 — FAQ / Tips（可选）🎨

**目的**: 减少评论区重复提问，体现专业度

#### 内容定义

| 要素 | 说明 |
|:-----|:-----|
| 常见问题 | 3-5 个最可能被问到的问题 |
| 简短回答 | 每个问题配 1 行简短回答 |
| 格式 | Q&A 卡片式排版 |

#### AI 生图提示词

```
设计一张FAQ信息图，竖屏3:4比例（1080×1440像素）。

顶部标题："❓ 常见问题解答"，36px加粗
下方排列3-5个Q&A卡片，每个卡片：
- 浅灰底（#F5F5F5）圆角矩形
- Q："{问题}"，加粗，深色
- A："{回答}"，常规，灰色
- 卡片间距16px

底部："还有问题？评论区告诉我 💬"

背景：纯白色
风格：干净清爽，信息层次分明
```
