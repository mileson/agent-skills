## 逐图详细规范

### 图 1 — 更新预告封面（必需）🎨

**目的**: 让老用户第一时间知道"更新了什么"，吸引新用户了解产品

#### 内容定义

| 要素 | 说明 | 示例 |
|:-----|:-----|:-----|
| 更新标题 | 重大更新的核心亮点 | "全新升级！四角独立圆角来了" |
| 版本号 | 可选，增加专业感 | "v2.2" / "重大更新" |
| 改进数量 | 数字钩子 | "3大新功能" / "5项优化" |
| 感恩语 | 拉近距离 | "感谢反馈❤️" |

#### 设计规范

| 属性 | 规范 |
|:-----|:-----|
| 尺寸 | 1080×1440（3:4 竖屏） |
| 构图 | 居中布局：App icon + 更新标题 + 亮点列表预览 |
| 配色 | 品牌主色 + 金色/橙色点缀（传递"升级"感） |
| 标题 | "🆕 {产品名} 重大更新"，48px 加粗 |
| 亮点预览 | 3 行带 emoji 的简短列表（24px） |
| 装饰 | 可加闪光/烟花 emoji 增强喜庆感 |

#### AI 生图提示词

```
设计一张产品更新公告封面，竖屏3:4比例（1080×1440像素）。

画面居中布局：
- 顶部：{产品名}的App图标（圆角正方形），旁边有"NEW"或"🆕"标签
- 中间：大标题"{更新标题}"，48px加粗，深色文字
- 标题下方：版本号标签"v{版本号}"，小字，圆角矩形背景
- 下方：三行更新亮点预览
  ✨ {亮点1}
  🎯 {亮点2}
  💫 {亮点3}
- 底部："向右滑动查看详情 👉"

背景：浅色底（{品牌浅色}），顶部有细微的金色渐变光晕（传递"升级"感）
风格：喜庆但不浮夸，专业但亲和
```

### 图 4 — 新旧对比（可选）🧩

**目的**: 直观展示改进幅度

#### 内容定义

| 要素 | 说明 |
|:-----|:-----|
| 左侧 | "旧版" 截图（标注版本号） |

### 图 5 — 用户互动征集（可选）🎨

**目的**: 增强社区参与感，收集下一版需求

#### 内容定义

| 要素 | 说明 |
|:-----|:-----|
| 感谢语 | "感谢大家的反馈❤️" |
| 下一步计划 | 简要透露后续开发计划 |
| 征集引导 | "你最想要什么功能？评论告诉我 💬" |

#### AI 生图提示词

```
设计一张用户互动征集图，竖屏3:4比例（1080×1440像素）。

画面布局：
- 顶部：大文字"感谢你们的反馈 ❤️"，40px加粗，温暖色调
- 中间：简笔画/扁平插图：一个开发者在电脑前看用户评论的场景，温馨可爱
- 下方：圆角矩形卡片，内含：
  "📋 下一版计划："
  "1. {计划功能1}"
  "2. {计划功能2}"
  "3. ???（你来决定！）"
- 底部："评论区告诉我你最想要的功能 💬"，28px

背景：暖色调（浅桃粉 #FFF0F0）
风格：温馨、亲近、社区感，让用户觉得开发者在乎他们的意见
```
