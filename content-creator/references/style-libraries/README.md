# 草稿风格库说明

## 目标

草稿风格库只负责定义“怎么写”，不负责提供事实、案例、历史表达素材。

内容来源仍然由 `content-creator-memory` 提供：
- 自有内容召回
- 标杆案例召回
- 选题相关的历史表达

风格库负责：
- 决定候选稿的切入角度
- 约束开头方式、结构方式和语气
- 明确禁止写法
- 帮助单平台并行生成 3 篇不同风格候选稿

## 当前固定风格

第一版固定为 3 种风格：

1. `technical-expert`
2. `storytelling`
3. `sharp-humor`

它们统一存放在 `core/` 目录，先跨平台复用。

## 使用方式

阶段4A时：

1. 主线程先完成当前平台的前置规划产物
2. 安装 3 个 writer subagents 到工作区 `.claude/agents/`
3. 主线程在 prompt 中显式传入：
   - 当前平台
   - 已确认事实与目标
   - 当前平台规则
   - 对应风格文档
4. 并行输出 3 篇候选稿
5. 主线程按固定表格在对话中输出推荐指数对比
6. 用户选中一个风格后，再展开为正式正文

## 编写原则

每个风格文档都应控制在一页以内，结构固定，避免写成大而空的风格论文。

必须包含：
- 定位
- 适用内容
- 不适用内容
- 开头方式
- 结构方式
- 语气要求
- 禁止写法
- 候选稿输出要求

## 禁止事项

- 不要把风格库写成另一个记忆库
- 不要在风格文档里存大量历史案例
- 不要让风格文档替代平台规则
- 不要在风格文档里发明阶段0-3之外的新事实

## 扩展原则

如果后续新增风格，优先判断：

1. 它是不是和现有风格有明显差异
2. 它能不能稳定写出不同的候选稿
3. 它是不是适合作为“候选草稿角度”，而不是最终精修风格

只有满足以上条件，才建议加入。
