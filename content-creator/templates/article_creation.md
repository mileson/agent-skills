# 任务事项：文章创作（Article Creation）

## 执行规则
1.  **严格遵循**: 严格按照下方的"待办事项"从上到下逐一执行，确保流程的完整性。
2.  **状态实时更新**: 每完成一个待办事项，立即更新其状态为 `[✅]`。当所有子任务完成后，父任务状态也应更新为 `[✅]`。
3.  **任务连续性**: 若任务中断，再次启动时需严格依据本文件的待办状态，从上一个未完成的任务开始，确保任务的连续性和目标一致性。
4.  **多重真理源**: 所有步骤必须以 `extracted_meta.json` 为主要指令，并以 `Memories/memory_indexing.json` 作为风格校准与内容增强的核心参考。严格遵循80/20策略：80%自有内容风格挖掘，20%标杆技巧借鉴。
5.  **路径约定**：所有文件操作均在 `Articles/<文章名称>/` 目录下进行。
6.  **使命终结**：本文件在成功生成并确认 `article_writing.md` 后，其使命即告完成。

> **流程概览**
> 1.  **智能记忆校准**：通过对 `memory_indexing.json` 的双层检索，80%挖掘自有内容风格，20%借鉴标杆技巧，建立混合风格基准。
> 2.  **生成大纲**：结合 `extracted_meta.json` 和混合风格基准，产出融合个人特色与吸引力技巧的 `article_structure.md`。
> 3.  **生成剧本**：将大纲拆解为包含完整执行规则的、原子化的 `article_writing.md`，明确80/20素材复用策略。
> 4.  **人工检查点**：大纲和写作剧本生成后，均需等待用户 `[确认]`。
> 5.  **流程交接**：一旦 `article_writing.md` 被确认，本文件使命完成。

## 待办事项

- [ ] **0. 智能记忆校准 (80/20 Strategic Memory Calibration)**
    - [ ] 0.1 **执行双层检索策略**：
        *   读取 `Memories/memory_indexing.json` 文件。
        *   **第一层检索（80%权重）- 自有内容风格挖掘**：
            - 基于 `extracted_meta.json` 中的 `target_audience`, `technical_stack`, 和 `seo.primary_keywords`，在 `own_works` 中执行精准匹配。
            - 优先级排序：`quality_score` (40%) + `readability_level` 匹配度 (30%) + 技术栈重叠度 (30%)。
            - **筛选出 Top 2 的自有高质量内容**作为"核心风格基准"。
        *   **第二层检索（20%权重）- 标杆技巧借鉴**：
            - 在 `reference_examples` 中，基于 `content_type` 和 `tone` 相似度，筛选出 Top 1-2 的标杆案例。
            - 重点关注其 `title_hooks`、`structural_patterns` 和 `expression_techniques`。
    - [ ] 0.2 **构建混合风格基准**：
        *   **主导风格提炼**：从 Top 2 自有内容中，分析并综合其 `tone`、`perspective`、`writing_style` 字段，确立"主导风格基准"（占80%权重）。
        *   **技巧增强提炼**：从标杆案例中，提取可借鉴的表达技巧和结构模式，作为"风格增强元素"（占20%权重），主要应用于标题与内容的创作。
        *   **生成融合指令**：将两者融合为一个完整的"智能风格基准"，明确标注主次关系。
    - [ ] 0.3 **提取分层素材库**：
        *   **核心素材（80%）**：从自有内容中提取 `reusable_elements`（包括`golden_sentences`、`core_workflows`、`success_factors`等）和 `key_metrics`。
        *   **技巧素材（20%）**：从标杆案例中提取可借鉴的标题钩子模式、开篇技巧、结构框架等。
        *   **智能组合策略**：记录如何在写作中优雅地融合这些素材，避免生硬拼接。

      ## 执行规则 (Execution Rules)

> **Tips for AI Agent**
> *   **80/20 记忆库策略**: 你的创作过程必须始于对 `memory_indexing.json` 的**双层检索**。80%专注于 `own_works` 的风格挖掘，20%借鉴 `reference_examples` 的技巧增强。检索的精准度（利用 `technical_stack`, `target_audience`）和质量筛选（利用 `quality_score`）是保证最终内容质量的基石。
> *   **智能风格融合**: 主导风格来自自有内容的深度分析（`tone`、`perspective`、`writing_style`），技巧增强来自标杆案例的表达技巧（`title_hooks`、`structural_patterns`、`expression_techniques`）。融合时确保和谐统一，避免风格冲突。
> *   **分层素材复用**: 核心素材（`golden_sentences`、`core_workflows`、`key_metrics`）来自自有内容，技巧素材（钩子模式、开篇技巧）来自标杆案例。在新语境下进行**创造性再利用**，而非简单复制。
> *   **质量控制原则**: 参考 [AI内容Google索引的6步法](https://ai.salesmarket.com/ai-content-6-steps-to-google-indexing.html) 和 [实体优化策略](https://on-page.ai/pages/entities/)，确保内容既有个人风格又符合搜索引擎优化要求。
> *   **执行节奏**: 大纲阶段不注入错别字；错别字与 Emoji 仅在后续具体写作阶段按指令引入。写作阶段务必同时遵循 `writing_style`、`humanization.style_quirks` 以及从记忆库校准的混合风格基准。
