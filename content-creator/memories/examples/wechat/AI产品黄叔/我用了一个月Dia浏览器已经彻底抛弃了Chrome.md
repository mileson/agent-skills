# 一.Dia的使用技巧

Dia的浏览器当前核心策略是AI Chat，因为是原生AI浏览器，它可以非常方便的获取到整个浏览器容器里的上下文，配合上现在强大的模型能力，已经稳定的风格控制，可以搞定非常多的问题，下面我举一些当前看到和总结下来的技巧，大家可以在这个基础上去延展：

## 1信息检索和内容延展

第一种最简单的玩法是，利用Dia对论文、热剧等内容进行快速解读，高效获取关键信息。

![Image](./assets/17516878126100.25248690834829335.png)

比如我们在看一个文章时，里面提到了冯国璋反对袁世凯，黄叔的历史不好，不知道其中的玄妙，直接问Dia，就能快速得到答案。

注意，此时的答案会结合网页里的内容综合给出，所以能很好地帮我们理解原文。

顺着这个思路，进一步的延展：

# 从Arc到Dia

Arc浏览器是我非常喜欢的浏览器，那它为何被“雪藏”呢？

![Image](./assets/17516878125450.8018062300985132.png)

从黄叔的角度，首先是Arc浏览器诞生的时机不对，在Chrome横行霸道二十年的情况下，没有颠覆式创新很难获得机会：

首先，它出生那会还没有AI，而有了AI后，Arc团队对于AI的迟钝性只能在原有基础上添砖加瓦，使得Arc仅仅是逐渐进化，而不是颠覆式创新的产物。

你可以理解为Chrome+Gemini也是同等逻辑的AI浏览器了，但都不是基于AI重新设计的产物。

其次，很多人喜欢的功能，事实上大众并不常用，Josh Miller在文章里写到：

只有 5.52% 的日活用户会经常用多个 Space。

只有 4.17% 用 Live Folders（包括 GitHub Live Folders）。

我们最喜欢的功能之一，日历悬停预览，只有 0.4% 的人用。

所以，Arc是很难走向大众市场，并且大部分用户也并没有更多的使用AI功能，于是Dia重新诞生了：

![Image](./assets/17516878126470.14036233919931396.png)

在Act II 宣传片的结尾，其实就展示了 Arc Explore 的原型，这个原型就是现在Dia的方向。

那Dia从前面我们介绍的技巧来看，核心是两个功能：Chat和Personalization

这两个功能分别有40%和37%的用户在用。

Chat前面我们都展示过了，Personalization的逻辑是设定AI如何回应你：

![Image](./assets/17516878132640.4536990146426654.png)

所以整个Dia产品的设计也非常简单，甚至很多人会问：这和[Monica](https://mp.weixin.qq.com/s?__biz=MjM5ODU1MzQzOQ==&mid=2451423412&idx=1&sn=c22e53e5cd7f237ee704c4259302397f&scene=21#wechat_redirect)（一个浏览器插件）有什么区别？

其实这只是个开始的起点，我们可以看到的是，Dia设计的时候就抱有一个理念：

简单留给用户，复杂留给自己。

黄叔测试下来，Dia对于多Tab的内容抓取是非常精准的，可以看出Dia在模型和工程方面做了非常多工作，包括可以接入YouTube。

以及，Dia也认为安全是重中之重。“为此，我们把安全工程团队从 1 人扩充到 5 人。我们投入红队测试、漏洞赏金和内部审计。”

这也是因为，后面会有更多AI Agent功能上线。

最后我想坦率地说：Dia 并不是对 Arc 及其不足的被动反应。不是。想象一下，在电灯诞生时，你写一篇文章解释为什么要放弃蜡烛生意。电智能已经到来——假装它不会从根本上改变我们需要打造的产品，是天真的。

希望更多人能用上Dia，这真的是黄叔的心头爱！

一款全新的AI浏览器，会从熟悉开始，似乎这是Chrome，但又有很多的不同，要做AI时代的人类，也要让自己果断地拥抱AI浏览器。
