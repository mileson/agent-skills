一、背景

前段时间看到一些领域大V在做 Cursor Rules （即 Prompt 指令）分享，乃至已经发
展到需要增加插件切换多个 Rules 。

其实一开始看到后，我有过想跟也做一个差异化更好用的，因为本身我也是懂得一些技
术的，我知道如何让这个事情更产品化 ：）

但是这几天想了下，如果往 Rules 做下去，其实对那些只是希望通过 Cursor 此类的工
具实现自己想法的人来说是个“陷阱”，大部分人其实“不配”用 Rules ，用不好的话，反

而会导致增加了更多不确定性的事情，产生更多的问题，还不知道怎么修复。

我并不否认 Rules 这个功能的价值，如果真的到需要精细化调优的话，那你可以考虑

下。

但是如果你也像我一样，由于刚学习 Cursor 进行 AI 编程，希望提高 AI 编程的效率与
效果，在网上找了很多 Cursor Rules 用于强化 AI ，但是总是得不到比较明显的效

果，那么，这篇文章就是为你量身定制。

二、为什么不推荐普通人使用 Cursor Rules

我们先来解释为什么使用 Cursor Rules 大概率得不到明显提升的原因：

（1） Cursor Rules 往往不具备个性化的差异

由于网上大部分能看到的 Rules 都是每个人在开发过程中根据个人编程习惯、认知沉淀

下来的经验，而每个人的习惯又有所不同。

小到有的人希望文档的目录结构是 A/B/C，而你希望的是 A/A1-A2，大到 iOS 软件开

发与安卓软件开发的区别。

如果你只是一名 iOS 开发者，你肯定不希望参考一个安卓开发者的 Rules ，因为可能
项目生态的规则不一样，即使 AI 编程不区分编程语言，但是编程语言的差异还是客观

存在。

如果你认为可以根据其他人的 Rules 进行改造，那你就会陷入到另外一个难题，即你可

能无法下手改动其他人的 Rules ，乃至越改越乱，因为你不懂，详见（2）的原因。

（2）Cursor Rules 设计需要具备专业能力

大部分人是不懂 AI 底层原理的，包括链式思维、预设角色等支撑的原理，况且也不需

要懂。

但是如果要设计一个富有成效的 Cursor Rules 是需要你熟悉无论是 Rules 的提供者

的编写的思考方式，还是 AI 大模型的原理，才有可能修改成属于自己的 Rules ，更别
说完全自己写一个 Rules 了。

这感觉就跟改别人的文章，或者程序员改别人的代码一样，请问，你还觉得稍微改下也

能用吗？

那么，如果 Cursor Rules 上手门槛这么高，那么是否有一种比较高效强化 AI 的万能

办法？答案是肯定的，这也是我写这篇文章的目的。

我从 Cursor 论坛的一篇求助帖子里，学到一个适用于大部分人的 AI 能力强化的技
巧，那就是通过 Cursor Docs 这个功能来实现强化 AI 编程能力。

论 坛 帖 子 的 地 址 ： https://forum.cursor.com/t/adding-new-docs-but-getting-deprecated-

code/36127

接下来，由我来为你一步步详细说明如何使用 Cursor Docs 提高 AI 编程效果。

三、如何使用 Cursor Docs

大部分的 AI 编程涉及到的内容，基本就是功能界面、功能逻辑、设计规范，此处以
iOS 编程为例，其他语言同理。

如果你是 iOS 开发者，那么你可以把 「Apple 设计规范」、「SwiftUI 文档（界面逻

辑）」、「算法模型文档」 的文档地址都添加到这个 Docs 内，Cursor 将自动搜索地

址内的文档材料并存档下来。

这样，后续在 iOS 开发过程中，Cursor 将默认优先使用文档内的知识库进行解答与编
程，直接按照官方最佳实践的案例进行编码，AI 开发的质量将得到大幅度提升，具体操

作步骤如下：

1、前往「Cursor 设置 - Docs 」功能，并点击「Add New doc」按钮，添加参考的

文档地址

温馨提示：iOS 开发专用文档，可关注本公众号并回复 “Cursor 01” 关键词免费领取

2、等待并确定 Cursor Docs 显示已添加的文档已被存档记录确保生效

⚠  注意：需要保证输入的链接最后有 / 符号，否则会出现只会查询到一个页面的情况

至此，只需要 2 步，我们就可以强化 AI 的编程能力啦，对于大部分人来说，通过这种

链接地址建立知识库的方法，简单、高效。

正如我之前的观点，我认为大部分人其实不应该把精力放到 Cursor Rules 这种高级的
AI的调优上，包括使用别人的Promt 可能会引入更多不可控的其他问题。

把大部分的精力放到使用工具上，工具调优的事情交给各个生态已有的最佳实践案例，

而非工具调优上才是最佳的实践方法。

所以，当我看到这个参考官方文档的用法时，我认为这才是所有新手或者普通创造者适
合的强化 AI 的方法。只需要给个参考地址，后续AI针对性的解决方案增强，也不会引

入更多不可控的要素。

(cid:0) 拓展下知识点，如果今年 Apple 发布新的技术，我们也可以把刚发布的文档地址也添
到 Cursor Docs 里面，让 AI 立即就能学会最新的技术并进行实现。

在其他人还在熟悉 Apple 新技术特性的情况下，你就可以直接让 AI 开发了，大概率会

让你拿到新技术的先发优势，赢在起跑线上～

四、总结

今天我们说的这种通过 Cursor Docs 文档引入的方式才是大部分人应该关注与持续学

习的，因为这真的很简单，只需要知道文档地址是什么，无论是官方的还是自建的在线

文档。

并且大部分人应该专注做一个领域的，即使多个领域，扩展知识库的概念也更符合人类
的习惯，而不是写一堆的 Rules 教 AI 要干什么，这不是你该做的事情。
